9:T1cb9,<p>悲劇了，每次都用不知道哪來的時間來寫文。</p>
<p>自從昨天很猛花了六小時專注寫程式後。</p>
<p>我的 Schedule 就被打亂。</p>
<p>原來厲害的人就是很忙很忙還能維持 Schedule 的人，學到了。</p>
<p>來紀錄一下昨天因為我聰明的小腦袋造就我得不要命六個小時拼命趕程式趕完。</p>
<p>原先起源於我前幾天又想幫我表姐買禮物。</p>
<p>可是我真的不知道要買什麼，因為 26 歲的女人了，送娃娃也不對。</p>
<p>送什麼也怪。</p>
<p>至於寫卡片，這種事情我做過很多事，可是，我知道當下很開心，過沒多久會被撒到一旁去。</p>
<p>所以有鑒於我自己寫程式，我就想說乾脆寫個 C 語言的 Hello World! 改成 Birthday.</p>
<p>給我的表姐點開跳黑視窗。</p>
<p>但，我又覺得這樣好像太 low 了，所以我就想說那就寫一個套版的 html，將 html 寄給他請他自己看。</p>
<p>這樣又覺得好 low，還得人家自己看。</p>
<p>於是我腦力激盪過後在路上想到之前練 django 教學有個暈端伺服器可以運用。</p>
<p>可是那個我不會用所以也是重頭學。</p>
<p>於是就趕緊搭上公車衝回家從 8 點半開始寫。</p>
<p>原先我想說 9 點半應該就完成然後做別的事。</p>
<p>沒想到我太天真了。</p>
<p>先從 Bootstrap 拉出一個美麗的套版。</p>
<p><img src="https://minayu0416.files.wordpress.com/2017/03/e89ea2e5b995e5bfabe785a7-2017-03-12-e4b88ae58d889-55-00.png" alt="螢幕快照 2017-03-12 上午9.55.00"></p>
<p>photo by bootstrap.</p>
<p>怎麼樣看起來超美的吧，很有質感。</p>
<p>然後根據自己想要的想像去改他的架構跟 css</p>
<p>剛開始就卡死在 css，因為我想要一張我跟表姐的照片滿版網頁。</p>
<p><img src="https://minayu0416.files.wordpress.com/2017/03/e89ea2e5b995e5bfabe785a7-2017-03-12-e4b88ae58d889-56-56.png" alt="螢幕快照 2017-03-12 上午9.56.56"></p>
<p>我硬是把網頁改成這個樣子。</p>
<p>剛開始我是把右上方的 list 刪掉</p>
<p>然後因為照片的關西，我只好把白色改成黃的，我個人覺得白的跟黑的會更有效果。</p>
<p>可是我媽堅持要黃色的。感覺一點都不搭啊....</p>
<p>後來發現加了字後把整個人都擋了。</p>
<p>加了我美麗的感言跟發自內心省斯的千字文後。</p>
<p>網站就擠不下了。</p>
<p>所以我就廢寢忘食(是這樣用嗎算了)的將我的千字文忍痛刪掉成百字文</p>
<p>這還不打緊，因為我打算讓他在手機上也能看的於是就變成這樣</p>
<p><img src="https://minayu0416.files.wordpress.com/2017/03/e89ea2e5b995e5bfabe785a7-2017-03-12-e4b88ae58d889-59-28.png" alt="螢幕快照 2017-03-12 上午9.59.28"></p>
<p>上面我沒照，簡言之就是字太多，版面塞不下。</p>
<p>然後我就在這樣的 css 排版跟 RWD 的摧殘下拉扯了兩三個小時。</p>
<p>到最後我覺得算了就這樣吧。</p>
<p>於是我就去研究那個雲端</p>
<p>叫做 pythonanywhere</p>
<p>原來這個要我把本來單單純純的 html 改成以 MVT 架構為主的後台。</p>
<p>真奇怪我怎麼記得我剛開始好像只是要做 html 而已</p>
<p>反正，我得扔到雲端註冊一個網址，再將網址扔給我表姐。</p>
<p>於是我就動手做。</p>
<p>從剛開始開終端機 key 虛擬環境，裝 python 然後創建 view, setting,....</p>
<p>反正這些東西以前在橋網站的時候都是學弟在搞的。</p>
<p>不料現在是我要用的只好搬出前陣子自學打得點根基。</p>
<p>然後偷偷連進學校網頁看看我當初怎麼寫的 XD</p>
<p>這部分大概耗了 30~1 小時。</p>
<p>最後兩小時(已經晚上 11 點了)</p>
<p>完全是在跟 pythonanywhere 奮戰</p>
<p>因為當初伺服器不是我架的</p>
<p>就算我把網站寫了差不多了，我還是不知道伺服器跟環境是怎麼架起來的。</p>
<p>於是我剛開始把 django 教學對 pythonanywhere 的步驟做一遍</p>
<p>於是我發現怪怪的，為什麼不能跑。</p>
<p>反覆懊惱了許久，在反覆回去確認步驟。</p>
<p>又把之後在雲端上寫的程式改改改，就是全錯，也不知道哪邊錯。</p>
<p>聰明的我就想到那就去 file 那邊看看，才發現我壓錯資料夾。</p>
<p>他的步驟是把你做好的 project 資料夾壓成一包，然後上傳到網站解壓縮。</p>
<p>我卡的點是在於我根本不知道原來 python 的驅動虛擬環境跟 project 資料夾是分開的！！！</p>
<p><img src="https://minayu0416.files.wordpress.com/2017/03/e89ea2e5b995e5bfabe785a7-2017-03-12-e4b88ae58d8810-12-33.png" alt="螢幕快照 2017-03-12 上午10.12.33"></p>
<p>birthday 是我開的 project</p>
<p>而 click 是我當初用虛擬環境去裝 python 這樣才能動。</p>
<p>然後就把我 2016 整年都在學的東西完全搞懂了，畫下一個句點。</p>
<p>早該劃下句點了嗚嗚</p>
<p>所以第一次我把名為 click 的 python 環境壓縮扔到網路上。</p>
<p>如果你把你的 project 上傳到網路上，你還需要在雲端上下指令創虛擬環境安裝 python</p>
<p>這樣才會有兩個資料夾。</p>
<p>原來我第一次在雲端上安裝 python 的時候就想說為什麼他說已經安裝好了。</p>
<p>原來根本就是自己傳錯檔案。</p>
<p>於是我抱著不肯放棄的心(原本好想放棄因為夜深了)</p>
<p>知道問題了這次我學聰明，我把 birthday 壓縮上傳到雲端上。</p>
<p>解壓縮後進雲端付的終端機，當時第二次我還不知道虛擬環境跟 project 是分開的。</p>
<p>於是我為了不要搞錯，我也把我在雲端上的虛擬環境取為 birthday。</p>
<p>一樣步驟，打開我的網站。</p>
<p>然後還是報錯。</p>
<p>第二個步驟我也是起笑了。</p>
<p>我的希望又泯滅了，此時我已經在想搞不定的話我要怎麼弄給我表姐看？</p>
<p>後來我堅持繼續在嘗試。</p>
<p>這次我對了，把 project 上傳到雲端。</p>
<p>然後在終端機裝虛擬環境取不同的名字。</p>
<p>分成兩個資料夾。</p>
<p>最後，打開我的網站，還是錯。</p>
<p>整個懊惱，失望。</p>
<p>於是，我覺得如果都照著步驟做，還是無法解決問題，我只好來看看他的紀錄到底報什麼錯。</p>
<p>我打開了 error 的報錯頁面，仔細一行一行看。</p>
<p>報錯只跟我提到一個問題。</p>
<p>拿去 google 後，回去雲端終端機改了一番。</p>
<p>最後終於完成了！！</p>
<p>在這次事情中我學到的除了要有信心堅持下去以外。</p>
<p>我發現自己從 2016/4-2016/10 搞了整整六個月的東西</p>
<p>我現在六個小時就能搞出來了。</p>
<p>最後，我還是想說，當你覺得照步驟照規局來不行。</p>
<p>那就跳脫步驟跟規局自己看自己動手做。</p>
<p>其實我取這個主題，主要是因為我常常有陣子好懶好懶，有陣子又昴起來工作</p>
<p>只能說轉變成工作模式的我真的超級超級投入跟認真。</p>
<p>對什麼都變得很認真跟很刁。</p>
<p>然後我要筆記兩個網站，待使用心得啦～</p>
