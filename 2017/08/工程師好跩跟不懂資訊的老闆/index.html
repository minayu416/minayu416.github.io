工程師好跩跟不懂資訊的老闆

August 30, 2017 minayu416

1 minute read

  • Twitter

  • Google+

  • Facebook

  • Reddit

  • LinkedIn

  • StumbleUpon

最近幾天我都會上網看「RD很跩」之類的文章

但我最近真的蠻受夠新來的資訊新手

後來我就終於知道為什麼網路上會「抱怨RD很跩」之類的

就像我這個剛開始什麼都還不太懂的小白兔

修煉成惡魔(喂～

工程師，為什麼我用Dreamwaver拉的asp不能在Django底下運作？
那我們用的伺服器是Linux還是Windows?
你看看這個網站！可以在Linux上用對不對！這怎麼弄？我就是想把我寫的asp扔進Django

新來資訊新手，以下就簡稱新人吧

剛開始進來還是以一種蠻跩的態度對我

所以我對他當然就是差不多的態度XD

不過最近有軟化了，大概知道我這個蛻變成惡魔的老屁股RD很難搞

開始用比較柔軟的語氣說「Python真的好難喔」(隱藏意義：你好強喔)

當然，也有可能是另外一種意義：「有什麼了不起哼」

至於為什麼RD們都這麼難搞呢？

這邊指的就是程式工程師喲

以下都是我猜測，因為我也是這樣過來的

女生工程師應該會好一點，男生會比較機車

  • 當他也是新手或者菜鳥的時候，比他還資深的老手幾乎都不理他，有問題都要他自己解決(google)，

久而久之，他就會不太想去理會新手，認為我以前都是這樣過來，為什麼你就不能自己google?

在這邊，不是只有我們這個小環境會這樣

因為我曾經有看過一個日本人新來的工程師分享他的經驗，他一進新公司，長輩工程師都不理他，也不跟他解釋在幹嘛。他就這樣度過了幾天完全不知道在做什麼，後來終於遇到有人肯跟他稍微解釋一下

而我現在不知不覺也會有這樣的習慣，覺得Google都有，你應該要學著自己找答案

  • 把伺服器或者網頁搞壞的時候，就會被罵

所以假設你是新手，一進來接觸然後又被工程師唸說自己去摸索，又不小心把伺服器搞壞問他怎麼辦，你一定會被罵，因為之前的老手也是這樣罵他

  • 大部分都是用責罵跟高壓力的方式逼著寫程式

搞壞就是被罵，但老手絕對不會跟你說如何不會搞壞(這點讓我好一陣子不敢寫Django的Model)

Code寫很差或很亂就是會被罵

不管做什麼都會被罵

  • 老手不屑理新手，因為老手的上一代老手也是這樣，被操的很強後都會覺得新人是腦殘跟智障，問一些智障問題，懶得回答

「我當初就是被罵被唸忍出來的，現在變成老屁股了，我懶得理你這些腦殘」，大概就是這個樣子

當我從一個新人準備要開始寫網頁的時候

我旁邊的學弟程式老手也是用這樣的方式對待我，如下

任何智障問題問他，他都會不屑跟不爽，然後叫你自己去google

寫壞任何東西會先被他唸/罵/嗆，不過幾乎都只是嘴砲無敵意

非 常 嘴 砲，然後會讓你自己覺得自己很爛很差

如何克服？

問別的態度好一點的工程師

一定會有天使存在，但是如果沒有天使工程師存在還有以下的方法

用良(ㄙㄚˇ)好(ㄐ一ㄠ )的態度詢問(女生佳)

像是盧他：「我真的不會啦～教我～～」「拜託啦！！我真的不會，我已經查詢好幾次了還是不懂」

裝傻裝呆「真的假的，你好厲害喔，懂這個真的好強＠＠」「我這個不太懂噎，是這個樣子嗎＠＠」

我的經驗是，我這樣做通常都會先被唸/嫌/嗆一番，然後工程師才會教你

我是不知道為什麼他們都要先嫌再教你

但是稱讚他還有把他捧高高久了，工程師就會內心暗爽，然後來教你這個腦殘

當然也是會有些工程師被稱讚久了變得更跩

也許是我已經撐過鑑賞期慢慢變厲害了

學弟終於慢慢對我刮目相看，並且變熟了

我現在都會有時向著學弟又柔弱又強勢打太極似的恩威並施

不能都居下風，還是要有個底線，萬一對方太超過，就兇一下，一定要有底線

大不了都自己學不要緊不用求別人

不管忍受多少來自老工程師的氣都要忍

你一定要心裡想著「等我以後能力起來了，你等著看」跟「等我變成老屁股了我也能欺負新人(喂～～～)」

稱過這個難熬的階段

恭喜你，慢慢和工程師親近了

根據我的學弟，就是一個很嘴砲的人，就常常跟他鬥嘴

然後稍微了解他們最近在學什麼在玩什麼

如果玩新的程式碼或者新的程式，有興趣的話也跟著學

反正新的程式碼跟程式學起來也不錯

所以我現在變成惡魔的老屁股

面對新來的一個新人也是想翻白眼

我們是用Django and Python

而新人是用Dreamwaver拉php, html, asp網頁

然後再叫我把他的網頁掛進Django

我是不知道asp能不能掛進Django

但這應該是兩種不同的語法才對吧？

掛上去了不太能跑，新人就開始丟各種苦惱的貼圖

(因為我先前已經有說過：我覺得php, asp不太能在Python下work)

所以發生這樣的事情我不太意外

但是是新人的部分網頁可能需要重做還是什麼的

他居然開始問說那伺服器是Linux還是Windows的？

因為他都用Dreamwaver拉網頁，自然就不會開終端機或者Putty下指令

也就不知道伺服器是Linux (我猜是asp 需要用Windows伺服器？)

所以問這樣的事情我應該可以斷定他應該是新手囉XD

然後開始找可以在Linux的伺服器底下Run asp的套件

要我幫他裝

我這時候當然很想翻白眼

  1. 我是兼差一個月才5000, 你是全職工作至少25k，然後要我騰出額外的時間幫你裝套件

  2. 為什麼不學學Python怎麼寫？ 你知道通常網頁工程師不太會用Dreamwaver拉網頁嗎，你要學的還多的QQ

所以我現在就是已讀他的訊息，完全不想理他，完全反映我這個變成惡魔的老屁股就是個活生生跩的工程師

不是啊，我才兼差，然後這個套件我不太會裝，我還要花時間去研究怎麼裝

我還要幫你寫後台…..

還有扛兩個案子跟我要開始研究生的下學期了

另外一件事就是不懂資訊的老闆或者HR

老闆應該不知道我們是用Django寫網頁，然後在外面隨便應徵到一個只要能寫網頁的人就可以了

老闆大概也不知道他應徵到的人是用什麼語法寫的，能不能夠合作，大概沒有講清楚

不過我猜新人也不太會跟老闆反映說用的語言不同

像是我是寫asp的，進來發現我們是用Python可能就要先有警覺需要學習新的語法

或者他可以堅持寫他的asp但他必須自己想辦法，因為我不會寫asp沒辦法幫他

至於為什麼不是我去學asp?

因為整個網站主架構都是用Django寫，而他的asp只是這個網站裡面的一個部分(新加進來的)

所以，剛開始懷抱程式熱忱的新手進來學程式通常都會遇到沒人理或者老手嫌自己笨

但撐過去後，經驗還有能力慢慢累積起來了，大家都會對你刮目相看

在我來說，目前如果有一個什麼都不會的新手來，我希望他有

  1. 肯學習有熱忱的良好態度

  2. 肯改變，像海綿一樣什麼都學，肯學習用不同的語法或者觀念

我就會想好好的教他

但是，一開始這位新手進來的態度就是一副什麼都不屑，什麼都不想學

只想用自己會的寫，不太願意做些改變或者學習新知識

因為我每次跟他溝通，他就覺得「我就是這樣啊」「我就是不想學」

還有比較上對下的眼神對我，所以我就沒有很多說什麼XD

老實說如果我以後去外面公司工作，假設公司希望我用別的語言的話，我也願意重頭學

最後

工程師一點都不呆喔！別看他們呆呆的看著你發呆

內心可是各種在思考怎麼解決這個Bug跟那個問題(我們老師就是)

會嘴砲的工程師大部分都覺得自己很聰明，你們是腦殘

還有有些工程師不善戀愛或者跟人交際

基本上我看一些文章有提到「聰明的人通常不太會社交」

就像是「The Big Theory」生活大爆炸/宅男行不行的Sheldon, 雖然是演的，但智商號稱180多

也是不太擅長社交

有些橋段會演到Sheldon用邏輯去推算怎麼跟人互動

但「聰明的人通常不太會社交」，這句話需要改一下

應該是說，理科拿手的人，通常邏輯能力較強

邏輯能力強的人會對於不能用任何邏輯解釋的東西感到困惑

像是

女人的心毫無邏輯可言

沒有辦法循著一個模式或者推算還有推測跟模擬出來該如何解決

因此，有可能是因為太重邏輯，所以比較不知道如何交際和交女友

跟女生相處會需要推算一些公式像是

如果女生生氣了大概的原因有以下

  1. 男生不理她

  2. 男生未讀她

  3. 男生已讀她

  4. 男生不體貼

結果是例外：大姨媽來

類似這樣的邏輯也許會在邏輯強的人的腦子裡打轉

真的不是在騙人，也許很難想像

但是，也許我也是邏輯強的人，所以有些比較夢幻的事情我時常用邏輯去思考

像是我就覺得「明天，我要和昨天的你約會」，這部日本電影據說很多人看了都想哭

但我在意的點只有為什麼會有平行時空？為什麼他跟她是倒著過生活，這樣的邏輯怎麼通

為什麼他覺得快樂的時候，女生覺得難過，為什麼他的第一次會是女生的最後一次

我是不太能想通這樣的事情，反而還會邊看整部片邊思考這樣的邏輯要怎麼去通

這樣看來，當大家哭得一塌糊塗的時候，我卻在思考一個越來越老一個越來越年輕這樣的邏輯完全不通

我的確在看這部片的時候，滿腦子如果男生是這樣的生活，那女生的生活怎麼推算？男生跟女生的生活怎麼推算怎麼有交集？

所以我完全哭不出來，加上下這種標題，結局大概很悲慘，我就不看了哈哈

你就不能拋棄邏輯來觀看這個唯美的電影嗎！！！！！

不行XDDDDD

現在我還殘留一點人性，不知道何時會變機械XD
