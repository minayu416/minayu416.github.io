不～專業教學文

週六

午後，是個令人舒適想睡的環境

不過還是被我的程式腦給喚醒

接觸Django應該差不多一年了

就是去年的現在

指導老師要我把我寫的php全部轉成Django

真是個噩夢

但我還是轉過來了

習慣了Django的模式後，我發現我比較喜歡這樣MVT(MVC)的模式了

想起今年四月時候接觸wordpress

然後那時候稍微架了一下也是用Python寫的CMS “Plone”

接著就被學長壓著學Wordpress

老實說沒有很喜歡被人家壓著學什麼

我比較喜歡在無壓力的環境下自己鑽研

有鑒於自己當初學的時候，被學弟臭唸一頓

不服輸的自己就來自己研究，然後發現了Django內建的後台

但有一次把model搞壞，我就有點害怕去接觸這塊

後來知道需要migrate/makemigrations

我就慢慢再回來研究這塊

但這樣簡簡單單的後台，背後是自己一個一個塞小Tag進去

也就是說要自己去設計該塞哪邊，該怎麼去轉換資料進html

因為畢竟我是先抄人家的網站Html再去想辦法做他的後台

跟現在的CMS，先做後面在設計前面是相反了

為了讓現在我正在幫學校設計的網站好用一點

目前趁沒辦指導老師抓走之餘，來鑽研這塊

本來這樣的時間應該是拿來唸英文的

但最近突然找回自己的道路後，就慢慢回來研究Python

不過發這篇文大概要讓人家失望了

因為我只會貼幾個網站

再慢慢去研究他XD

https://www.django-cms.org/en/

http://mezzanine.jupo.org/

雖然我有時候會覺得我們老師為什麼不乾脆直接交CMS

不過，我還是很感謝他讓我從前端包後端再包資料庫

也就是全端都操練一次XD

聽說這個網站，有Python的開發編譯器

https://bitbucket.org/

可是身為開發網站的自己，我覺得Atom已經很好用了

也有人推薦Gedit，目前大概只先載下來，有招一日再來玩玩XD

至於Sublime嗎？

好像需要錢，我就沒有去使用了

最後，也來記錄一下

有次刷臉書，跳出來的一個廣告網站

是有關於教學Python跟Data Science的網站

https://www.datacamp.com/courses/intro-to-python-for-data-science?utm_source=fb_paid&utm_medium=fb_desktop&utm_campaign=dc_visitors

還有如果要學Git的話，這邊也有外面大大在教學的網站唷XD

http://tech-marsw.logdown.com/blog/2013/08/16/git-notes-github
