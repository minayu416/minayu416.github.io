架Wordpress on Amazon Could遇到的問題

December 26, 2017 in IT Things, [BLOG] 關於部落格的那點事

在Wordpress裡，要載插件時，會遇到Wordpress需要進入Web Host的狀況

在這邊我卡了超 級 久

因為AWS之前我都是用SFTP連

他就沒有FTP QQ怎麼連

任何Instance的Security Group都設了

這個問題是當你架好Wordpress也順利創好後，你第一次要載套件或插件跟主題，系統會通知要計入你的Host Web 麻煩你輸入FTP的使用者名稱跟密碼

而我在AWS都是用SFTP去連

到最後我找到這個

sudo chown -R apache:apache /var/www/html

需要開始起終端機或Putty連進你的雲端server，然後輸入這行

之後就可以了！

另外，我之前還卡住幾個點

像是當我要創Wordpress的Database跟創wp-config.php的時候也是吃盡苦頭QQ

如果是用AWS的RDS資料庫的話，記得要去

RDS >> Security Group >> inbound 將本身就有的協定加上一個自身(手動新增rds instance的協定)打開成Anywhere

這樣可以讓wordpress與RDS的資料庫連結

另外，當環境架好跟Wordpress裝好，要進入安裝Wordpress的時候

系統會詢問你的資料庫資訊，用來創wp-config.php檔

可是，因為系統創不了，他有給我範例，讓我自己去創

wp-config.php 我是用終端機手動輸入的

在wordpress解壓縮後會有一個wp-config-simple.php檔

在裡面打上資料庫的一些資訊

像是DataBase name, User Account, User password, Host.

然後存起來，再改名字為wp-config.php

這樣回去安裝wordpress的頁面，他就會run下一頁了

我覺得整體來說wordpress沒有很穩….

當然…我不知道是我租的主機不穩還是wordpress不穩

不然就是根本我自己不太會用，沒什麼設定到

不過我也是初學者QQ

只希望這次重裝好後，不要再出太大的問題

最新卡的問題是沒辦法把現在在這的文章匯進新架好的wordpress裡

而且這麼不穩定我也不敢轉戰….

不過說真的，在Wordpress.com註冊網站，雖然限制有點多(對一般民眾已經非常夠用了)

自架Wordpress可以享受盤古開天的感覺

有工程師跟公司幫你管理你的網站是好事，這點千萬要記住，雖然限制挺多，但限制的目的就是控制使用者不要亂改東西造成網頁掛點，同時也創一個可以讓使用者既可以不要動到深層的程式但還是能夠自由的裝飾自己的網站

但，如果是盤古開天自己架Wordpress.org的話，網頁掛掉什麼的，都是自己負責

不用用到“負責”兩個字這麼的重，但就是網頁掛了沒人給你掛保證

什麼都得自己來，當然啦自己架網站就是得先任命什麼都是從0開始

所以什麼套件什麼功能都得自己載自己弄

你能想像到多陽春就多陽春

當然使用Wordpress本身就是幫你寫好很多，也不會陽春到很陽春(你到底在說什麼)

注意

一個不小心會把網站弄掛

如果你嘗試跟著搜尋到的網頁，在你的終端機上跟著輸入指令

可能就會掛點

因為我有掛過，以致於我現在超級小心XD

不過我這網站是在聖誕節架的，希望是好的聖誕節禮物(就是別再掛啦QQ)

好處？

可用的主題超多XD，可自訂樣式改CSS

還有我如果給我的DataBase超多容量

那我的影音就可以傳到那樣的容量

像是我現在有時候寫遊記配上很多照片快把公司分配給我3GB的容量給用完了

很多主題＋可自己自訂

我可以自己分配多少容量來給我上傳影片跟圖片寫遊記跟日記

這兩個功能就夠我想自己架站了XD

不然在這個網站更新遊記的話，應該一下子就…..滿了XD

目前正測試新架網站的穩定性，還有摸索功能跟操作

穩定了，才敢把遊記跟文章更新上去

如果真的不錯，穩定也有意經營下去的話，我應該會等明年錢進來後，租AWS的雲端

除了開發網頁，還可以開發其他東西
