架Wordpress on Amazon Could遇到的問題

Posted on Dec 26, 2017

在 Wordpress 裡，要載插件時，會遇到 Wordpress 需要進入 Web Host 的狀況

在這邊我卡了超 級 久

因為 AWS 之前我都是用 SFTP 連

他就沒有 FTP QQ 怎麼連

任何 Instance 的 Security Group 都設了

這個問題是當你架好 Wordpress 也順利創好後，你第一次要載套件或插件跟主題，系統會通知要計入你的 Host Web 麻煩你輸入 FTP 的使用者名稱跟密碼

而我在 AWS 都是用 SFTP 去連

到最後我找到這個

sudo chown -R apache:apache /var/www/html

需要開始起終端機或 Putty 連進你的雲端 server，然後輸入這行

之後就可以了！

另外，我之前還卡住幾個點

像是當我要創 Wordpress 的 Database 跟創 wp-config.php 的時候也是吃盡苦頭 QQ

如果是用 AWS 的 RDS 資料庫的話，記得要去

RDS >> Security Group >> inbound 將本身就有的協定加上一個自身(手動新增 rds instance 的協定)打開成 Anywhere

這樣可以讓 wordpress 與 RDS 的資料庫連結

另外，當環境架好跟 Wordpress 裝好，要進入安裝 Wordpress 的時候

系統會詢問你的資料庫資訊，用來創 wp-config.php 檔

可是，因為系統創不了，他有給我範例，讓我自己去創

wp-config.php 我是用終端機手動輸入的

在 wordpress 解壓縮後會有一個 wp-config-simple.php 檔

在裡面打上資料庫的一些資訊

像是 DataBase name, User Account, User password, Host.

然後存起來，再改名字為 wp-config.php

這樣回去安裝 wordpress 的頁面，他就會 run 下一頁了

我覺得整體來說 wordpress 沒有很穩....

當然...我不知道是我租的主機不穩還是 wordpress 不穩

不然就是根本我自己不太會用，沒什麼設定到

不過我也是初學者 QQ

只希望這次重裝好後，不要再出太大的問題

最新卡的問題是沒辦法把現在在這的文章匯進新架好的 wordpress 裡

而且這麼不穩定我也不敢轉戰....

不過說真的，在 Wordpress.com 註冊網站，雖然限制有點多(對一般民眾已經非常夠用了)

自架 Wordpress 可以享受盤古開天的感覺

有工程師跟公司幫你管理你的網站是好事，這點千萬要記住，雖然限制挺多，但限制的目的就是控制使用者不要亂改東西造成網頁掛點，同時也創一個可以讓使用者既可以不要動到深層的程式但還是能夠自由的裝飾自己的網站

但，如果是盤古開天自己架 Wordpress.org 的話，網頁掛掉什麼的，都是自己負責

不用用到“負責”兩個字這麼的重，但就是網頁掛了沒人給你掛保證

什麼都得自己來，當然啦自己架網站就是得先任命什麼都是從 0 開始

所以什麼套件什麼功能都得自己載自己弄

你能想像到多陽春就多陽春

當然使用 Wordpress 本身就是幫你寫好很多，也不會陽春到很陽春(你到底在說什麼)

注意

一個不小心會把網站弄掛

如果你嘗試跟著搜尋到的網頁，在你的終端機上跟著輸入指令

可能就會掛點

因為我有掛過，以致於我現在超級小心 XD

不過我這網站是在聖誕節架的，希望是好的聖誕節禮物(就是別再掛啦 QQ)

好處？

可用的主題超多 XD，可自訂樣式改 CSS

還有我如果給我的 DataBase 超多容量

那我的影音就可以傳到那樣的容量

像是我現在有時候寫遊記配上很多照片快把公司分配給我 3GB 的容量給用完了

很多主題＋可自己自訂

我可以自己分配多少容量來給我上傳影片跟圖片寫遊記跟日記

這兩個功能就夠我想自己架站了 XD

不然在這個網站更新遊記的話，應該一下子就.....滿了 XD

目前正測試新架網站的穩定性，還有摸索功能跟操作

穩定了，才敢把遊記跟文章更新上去

如果真的不錯，穩定也有意經營下去的話，我應該會等明年錢進來後，租 AWS 的雲端

除了開發網頁，還可以開發其他東西
