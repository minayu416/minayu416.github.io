軟體開發與測試

第一堂課

MVC

Struts ，用Java寫的

open source，web application，鼓勵開發者去使用

MVC = Model + View + Control

專門做Web application 

url上面 會是一堆16進位，2進位的亂碼

但為了造福廣大使用者，他們會轉成中文。

瀏覽器第一個要做的：
將網頁名稱(dcetit.files.wordpress.com)轉換成 ip位置(這電腦才看得懂)

https：網頁通訊協定

A set of rule is called a protocol

hypertext transmission protocol Secure

也被稱為HTTP over TLS，HTTP over SSL或HTTP Secure

兩個人之間講好的見面規則

hypertext 用文字送圖，用文字送聲音。

所以要把圖跟聲音轉成文字，這東西叫做”編碼“

鍵盤打得出來看得出來的字才能送，也就是說中文字得轉換成編碼。

https://
TCP
port number  :80⁄443
打個比方，沒加s就是明信片，有加s就是信封
TCP要打電話，撥接
＃電話服務
＃TCP補IP的不足，要補保證你送的東西一定會收到，所以他會回送封包。
＃他會照順序送
Https在TCP上面
送不到對方的時候就是斷線。
www.google.com.tw/
IP
每張網卡都有一張ip
wifi也有一個，只要有ip就會有一個收跟送的動作。
ip是明信片
＃信件服務

-實體層
-應用層
-連接層
-網路層
-傳輸層

IP的規則，建一個全世界的物聯網讓任何一個人可以寄東西給任何一個人所以他把郵局的地址改成2的16次方大的ip

打太極拳的送法，以台灣為例，若要寄給中國就往左邊的信箱丟，寄給日本往上，寄美國往右。
丟給別人幫我寄
所以丟給“路由器”，要丟的地方需要有兩張以上網卡。

區域網路:自己寄
所以如果是自己能寄到的，得自己寄，叫做區域網路。

就像信箱：綠信箱分成本地跟外縣市。

LAN是一步送到，那WAN就是對外連網路。
**
**＃IP不穩定，不保證送到對方那。
＃有可能，你送123收到變成321。
**
**那為什麼兩個以上的連線可以建立在一張網卡，那就是用port number去接。

但有保留的port number

一個ip有2的16次方個連線。

也就是說，在家開網咖都ok。

懶人包：一個要連線一個不用連線。

**webhp?
這寫法的就是MVC
action

網址有看到 .php 就是圖法煉鋼的網站，不是用MVC架構。
.asp 類似這種就是沒有架構的想法，暴力法寫的網頁。

那….什麼是MVC?
一個網站哪些事長得一模一樣的。
那就是瀏覽器上方的縮小，叉叉，放大跟nav列….之類每個視窗都一定會有的，這叫做套版。

sourceid=chrome-instant&

最早的架構

大家，乖乖學看英文吧。看久了就會變成中文的。

抓database的過程，是看不到的，也就是說後台的東西是看不到的。

你只看得到Request跟Response的東西。

早期的日期是自己寫的，可是怕大家會輸入沒有的日期，解決的方法就是，下拉式選單。

Data Manipulation 就是要遮掉幾個字或者幾些東西來保護。

Model 1 就是圖法煉鋼的模式

HTML Tutorial 

Hypertext Markup Language

HTML教學，裡面也付CSS,JS,PHP….之類的內容。

基本上可以邊寫網頁邊看。

不要用閱讀文章的方式去看這個Tutorial，雖然我一開始學是就是用閱讀的方式去背….

JavaScript

基本上就是處理網頁上所有的動作。

瀏覽器只做：1.根據HTML畫網頁 2.動態網頁 JavaScript 的執行

CSS

幫你的網頁裝飾跟穿衣服。

一個網頁分為前台跟後台，以上的語法都適用於前台，也只用於前台。

後台的部分下回揭曉。

那這學期不寫 Model 1，而是寫 Model 2。
