<p><code>使用工具: Mac, Python 3, Terminal</code></p>
<p>從開發程式開始，我就堅持用 mac，雖然在研究所時期被學弟跟指導老師唾棄嘲笑過，因為學校都用 windows，導致我自己用 mac 水土不服，所有跟 mac 相關的系統問題都要自己研究</p>

<p>mac 系統很像 Linux, 所以直接學 Linux 也可以</p>
<hr>
<p>對於一個 python 新手，安裝環境會卡很久, 尤其是在你甚麼都搞不懂的情況下</p>
<p>我自己剛進公司，剛買一部電腦，裝環境時也是很頭痛</p>
<p>你可以選擇用</p>
<ul>
<li>Anaconda</li>
</ul>
<p>https://www.anaconda.com/</p>
<p>這個好處就是套裝， 一套走天下</p>
<p>壞處就是你會沒經歷過混沌亂搞時期，沒有體會神經元斷開重接的樂趣</p>
<p>自然會不知道如果沒有 Anaconda 時，你該怎麼架好你的環境</p>
<ul>
<li>自己手動裝</li>
</ul>
<p>https://wsvincent.com/install-python3-mac/</p>
<p>我想教學都寫得很清楚</p>
<p>我自己買新電腦是用 Anaconda 裝</p>
<p>現在就開始吧，我就直接講第二個自己手動裝了</p>
<p>打開你的終端機 Terminal，你可以在 mac 應用軟體內找到</p>
<p><img src="https://minayu0416.files.wordpress.com/2016/07/terminal.png" alt="terminal"></p>
<p>一開始，你的目錄會在 User/ (應該會是你取的名字)</p>
<p><strong>也就是你的 Home, 記好這個位子會是 Home, 因為以後你找英文網站，他會請你前往你的 Home 就是這一層</strong></p>
<p>你可以使用</p>
<pre><code>$ ls
</code></pre>
<p>來查看這一層目錄有哪些檔案跟資料夾</p>
<p>通常如果你裝 Anaconda, 他會幫你裝在這一層</p>
<h3 id="安裝-xcode-and-homebrew"><strong>安裝 Xcode and Homebrew</strong></h3>
<p>安裝 Python，我們會需要 Mac Xcode 的套件管理 Homebrew</p>
<p>(只要複製$ 以後的指令到終端機即可)</p>
<pre><code>$ xcode-select --install




$ ruby -e “$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
</code></pre>
<p>確認您的 Homebrew 安裝好了</p>
<pre><code>$ brew doctor
Your system is ready to brew.
</code></pre>
<hr>
<h3 id="安裝-python-3"><strong>安裝 Python 3</strong></h3>
<pre><code>$ brew install python 3
</code></pre>
<p>確認 python 3 安裝好了</p>
<pre><code>$ python3 --version
Python 3.7.0
</code></pre>
<p>打 python 3 就可以直接用 python 來編譯了</p>
<pre><code>$ python3
...
...
Type "help", "copyright", "credits" or "license" for more information.
>>>
</code></pre>
<p>使用 exit() 離開 python 3 環境</p>
<pre><code>exit()
</code></pre>
<hr>
<h3 id="安裝-pip"><strong>安裝 PIP</strong></h3>
<p>這個非常重要</p>
<p>pip 指令是 python 的套件安裝與套件管理工具</p>
<p>使用 python 後，你將會有極大部分機率常常使用 pip</p>
<p>python 本身環境沒有 pip，除非有方法可以開虛擬環境(開了一個虛擬環境 pip 會幫你安裝好!)</p>
<pre><code>sudo easy_install pip
</code></pre>
<p>*<strong>* 不過好像建立一個虛擬環境，環境就會幫你把 pip 安裝好了，所以也不用擔心，要真的不行再回頭來安裝 pip 吧**</strong></p>
<hr>
<h3 id="建立虛擬環境"><strong>建立虛擬環境</strong></h3>
<p>當我們要執行一個新的 project 或軟體開發時，我們會安裝一個虛擬的環境來做開發</p>
<p>通常會在 Home 目錄下開一個新的虛擬環境</p>
<p>但我個人喜好是在桌面 (所以一進公司就被咱偉大的 Senior 前端唸了一番哈哈 xD)</p>
<p>後來也漸漸的在我的 Home 目錄底下開一個 project 資料夾在裡面放我的開發虛擬環境</p>
<p>虛擬環境就是你在裡面亂裝甚麼，怎麼搞破壞，只要把環境的資料夾砍掉，也不會影響到電腦</p>
<p>缺點就是.....</p>
<p>萬一你每次打開電腦回來開發時，忘了 run 虛擬環境，你還是會把一堆雜七雜八的套件不小心安裝在你的電腦 xDDD</p>
<p>手動開啟虛擬環境</p>
<pre><code>$ ls
</code></pre>
<p>確保你在你在 Home 裡</p>
<p>資料夾應該會有</p>
<p>Desktop, Download, Application..</p>
<p>如果你安裝 Anaconda, 他也會在這一層</p>
<p>如果你使用 conda 指令來開啟專案</p>
<p>專案資料夾會在 anaconda 裡</p>
<p>開啟一個新的資料夾，你可以取名為 project，也可以取為任何你想要取名的名字</p>
<pre><code>$ mkdir project
</code></pre>
<p>然後切換到裡面</p>
<pre><code>$ cd project
</code></pre>
<p>在這裡面你會放一堆你開發的專案，接著我們就開始，開啟一個虛擬環境</p>
<p>通常你會再創一個資料夾，就是你的專案名字</p>
<pre><code>$ mkdir mysite
</code></pre>
<p>切換進去你的專案資料夾</p>
<pre><code>$ cd mysite
</code></pre>
<p>所以這個時候你應該會在這</p>
<pre><code>User
    Desktop
    Download
    Application
    project
        mysite &#x3C;&#x3C;
</code></pre>
<p>這個指令是，在你目前的位置裝一個虛擬環境</p>
<pre><code>$ python3 -m venv .
</code></pre>
<p>這時候你的 mysite 資料夾裡應該會多出一些檔案</p>
<pre><code>bin
lib
...
</code></pre>
<p>接著將虛擬環境 run 起</p>
<pre><code>$ source bin/activate
(mysite)$
</code></pre>
<p>這個時候你在環境內亂搞，才不會影響到電腦</p>
<p>然後離開你的虛擬環境</p>
<pre><code>(mysite) $ deactivate
$
</code></pre>
<p>＊＊＊＊為什麼我在這要用 mysite，其實大部分教學的虛擬環境都會是</p>
<p>ooo_venv/ xxx_env 為你的虛擬環境名稱，是為了方便初學者釐清觀念</p>
<p>不過我們做專案跟工作時，我們是會先用 Git 把 GitHub 上的專案 Clone 下來</p>
<p>然後在切換進專案資料夾內安裝虛擬環境</p>
<p>這也是我開始工作後才發現原來工作上的前輩們都是這樣</p>
<p>假設 project 叫 food_web</p>
<p>我們會切換進 food_web 內，然後在 run</p>
<pre><code>$ python3 -m venv .
</code></pre>
<p>＊＊＊＊</p>
<p>用 Anaconda 的人會使用 conda 指令開一個虛擬環境</p>
<p>因為 conda 可以直接指定版本要 2 還是 3</p>
<p>對於我這個之後才學 3 的人，前陣子不清不楚</p>
<p>最後才學會開環境 2 跟環境 3 的人，經歷了一段渾沌練功時期 qq</p>

<h3 id="安裝-django"><strong>安裝 Django</strong></h3>
<p>這部分就推薦</p>
<p>https://djangogirlstaipei.gitbooks.io/django-girls-taipei-tutorial/django/installation.html</p>
<p>我的程式幼幼班也是參考他的 QQ</p>
<p>現在我們已經在虛擬環境內了.</p>
<pre><code>(mysite)$
</code></pre>
<p>那就開始照網站的內容下指令吧!!</p>
<p>使用我先前說的，你一定會大量使用他的 python 套件 pip 來下載 django</p>
<pre><code>(mysite)$pip install "django&#x3C;1.9"
</code></pre>
<p>(我是不知道為什麼網站要侷限 django &#x3C; 1.9</p>
<p>我自己的話是不會做這種事，當然後果可能就是，萬一你用 1.9 以上的版本，有其他問題要自己 django 文件看內容 qq</p>
<pre><code>Installing collected packages: django
Successfully installed django-1.8.6
</code></pre>
<p>代表成功安裝好了</p>
<p>接著，就開始開一個新的 django app 吧!</p>
<pre><code>&#x3C;code>(mysite)$ django-admin.py startproject nuuk&#x3C;/code>
</code></pre>
<p>我承認這是我亂搞 xD，因為我真的不知道要取甚麼名字對於專案內容</p>
<p>nuuk 是格陵蘭的城鎮，你可以取任何業主或者你自己想叫的名字</p>
<p>目錄會長這樣</p>
<pre><code>nuuk/
─ manage.py
└── nuuk/
    ├── __init__.py
    ├── settings.py
    ├── urls.py
    └── wsgi.py
</code></pre>
<p>接著下指令將伺服器 run 起來</p>
<p>在有 manage.py 檔案那一層</p>
<pre><code>$python manage.py runserver
...
...
Starting development server at http://127.0.0.1:8000/
Quit the server with CONTROL-C.
</code></pre>
<p>這個就是所謂在<strong>本地端</strong>或者<strong>local 端</strong>做測試跟開發</p>
<p>打開你的估狗輸入</p>
<p>http://127.0.0.1:8000/</p>
<p>就可以看到有頁面 show 出來</p>
<p>當然我們這些人會使用別的指令 xD，我個人最愛怪怪的數字</p>
<pre><code>$python manage.py runserver 0.0.0.0:9912
</code></pre>
<p>http://127.0.0.1:9912/</p>
<p>80 port 才是對外開放，除了避開一些特定數字</p>
<p>不然其他數字 port 都可以取，不過僅限開發跟測試用</p>
<p>正式對外開放還是 80 port 居多</p>
<hr>
<h3 id="django-applicationapp">Django application（app）</h3>
<p>最後建立 app</p>
<p>Django 是 MVC/MVT 架構，所以內容編寫會在 View.py 與 Model.py 居多</p>
<pre><code>$ python manage.py startapp trips
</code></pre>
<p>我就直接照網站上 key 了</p>
<p>一樣你可以依照你開發的功能去取 app 名字</p>
<p>然後記得在<em>settings.py 加上你新安裝的 app</em></p>
<pre><code># nuuk/settings.py

...

# Application definition

INSTALLED_APPS = (
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'trips',
)
</code></pre>
<p>MVC / MVT 的</p>
<p>View.py 跟 Model.py</p>
<p>會在你的新 app 裡(trips)</p>
<p>所以你可以在你的 nuuk 專案裡開很多 app</p>
<p>然後在各種不同的<strong>import</strong></p>
<p>至於 C/T 的 Template</p>
<p>大家就去估狗啦，基本上會需要手動建立 Template</p>
<p>位置應該是 app 的上一層</p>
<p>Template 裡面會放不同的 html/css/js</p>
<p>之後探索的樂趣就交給各位啦</p>
