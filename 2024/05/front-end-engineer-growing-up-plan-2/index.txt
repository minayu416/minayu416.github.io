<h2 id="html-cheatsheet">HTML Cheatsheet</h2>
<p>其實基本上就是照著這個<a href="https://htmlcheatsheet.com/">Cheatsheet</a>的內容複習一次，以下列舉一下我自己個人覺得會忽略或者不知道的要點。</p>
<h3 id="img">Img**</h3>
<pre><code class="language-html">&#x3C;img src="/demo.jpg" alt="baby" longdesc="desc.txt" width="100" height="48" />
</code></pre>
<ul>
<li>img tag 有直接的 height 跟 width attribute，我之前寫 css 都是直接寫 class style，但老是忘記 html 可以直接寫高寬的屬性。</li>
<li>img <code>alt</code> 和 <code>longdesc</code>，<code>alt</code> 就是對於圖片的簡短的文字描述。<code>longdesc</code>據說是對於圖片比較長的文字描述，參考文件的描述如下
<ul>
<li><code>&#x3C;img></code> 標籤的 <code>longdesc</code> 屬性與 alt 屬性類似，但它允許更長的描述性文字。longdesc 的值是一個指向包含影像說明的文件的 URL。如果說明多於 1024 個字元，那麼可以使用 longdesc 屬性來設定指向它的鏈接。</li>
<li>可是呢目前瀏覽器都還沒有支援 longdesc 屬性。因此，對於如何建立那些大塊的描述，我們的建議是建立一個超鏈接指向包含描述的頁面即可。</li>
</ul>
</li>
</ul>
<h3 id="a">a</h3>
<pre><code class="language-html">&#x3C;a href="https://HTMLforBabies.com/" target="_blank" rel="external nofollow"
  >Link&#x3C;/a
>
</code></pre>
<ul>
<li>再來是 <code>&#x3C;a></code> tag 的屬性 <code>rel</code> and <code>rev</code>, 至從開始調查 SEO 後，我發現 <code>rel</code>這個屬性是其中一個優化 SEO 的要點，他能夠告訴機器人爬蟲這個連接跟現在看到的網站是什麼關係，不過詳細還沒有使用過，未來會嘗試看看。</li>
<li><code>target</code>, 可以選擇點選網站時，會怎麼打開這個網站，比方說打開連結於開啟新的視窗(_blank)、佔整個視窗(_top)，還有其他。</li>
<li><code>hreflang</code>, 可以指定或者說提示連結的網站是屬於哪個語言。</li>
</ul>
<h3 id="robotstxt">robots.txt</h3>
<p>robots.txt 是一個放置在根目錄的文檔，他主要是讓機器人爬蟲知道這個網站能不能訪問。機器人會爬蟲你的網站後，在他們的資料庫建立關於你的網站的索引與資訊，接著會決定你的網站能不能被人在搜尋引擎上找到 (SEO)。</p>
<p>案例</p>
<pre><code>User-agent: *
Disallow: /dont-index-this-folder/ &#x3C;- 在這個目錄後的檔案都不要訪問
Sitemap: https://htmlcheatsheet.com/sitemap.xml &#x3C;- 類似告訴機器人關於這個網站的地圖／導航，通常會另創另一個sitemap.xml檔案並由 robots.txt指定。
</code></pre>
<p>禁止訪問</p>
<pre><code>User-agent: *
Disallow: /
</code></pre>
<p><code>robots.txt</code> 跟 <code>sitemap.xml</code> 是我開始在經營這個個人網站，研究 SEO 後發現很重要的兩個檔案，若您的網站有需要 SEO 優化的話，可以研究實作這兩個檔案。</p>
<h3 id="html-5">HTML 5</h3>

&#x3C;article>&#x3C;/article>
&#x3C;section>&#x3C;/section>
&#x3C;main>&#x3C;/main>
&#x3C;aside>Sidebar&#x3C;/aside>
&#x3C;address>&#x3C;/address>
&#x3C;footer>&#x3C;/footer>
</code></pre>
<p>以上這些是 HTML 5 才出來的 tags。</p>
<h3 id="div">div</h3>
<pre><code class="language-html">&#x3C;p dir="rtl">
  This paragraph is in English but incorrectly goes right to left.
&#x3C;/p>
&#x3C;p dir="ltr">This paragraph is in English and correctly goes left to right.&#x3C;/p>

&#x3C;hr />

&#x3C;p>هذه الفقرة باللغة العربية ولكن بشكل خاطئ من اليسار إلى اليمين.&#x3C;/p>
&#x3C;p dir="auto">
  هذه الفقرة باللغة العربية ، لذا يجب الانتقال من اليمين إلى اليسار.
&#x3C;/p>
</code></pre>
<ul>
<li>div 的 dir 是我在開發 Next.js 時挖到的一個屬性，<code>dir</code> 可以指定 div 裡的方向， <code>rtl</code> 就是靠右、<code>ltr</code>靠左、還有其他值。</li>
</ul>
<h3 id="atribute">atribute</h3>
<p>HTML Tags 裡有很多不同的屬性，可以多參考 Cheatsheet 內的 Attributes 了解更多。</p>
<h3 id="head-tags">Head Tags</h3>
<pre><code class="language-html">&#x3C;head>
  &#x3C;meta charset="utf-8" />
  &#x3C;!-- 這一行是瀏覽器適配性，特別是針對 IE，因為並不是所有的網站都能在 IE運行，這一行主要是設定這個網站要用ie哪個版本渲染網站 -->
  &#x3C;meta http-equiv="x-ua-compatible" content="ie=edge" />
  &#x3C;meta name="viewport" content="width=device-width, initial-scale=1.0" />
  &#x3C;link rel="canonical" href="https://htmlcheatsheet.com/" />

  &#x3C;!-- 這些算是SEO的一部分，就是設定這個網站的名字、描述跟關鍵字，雖然先前的文章有說Google已經不會參考關鍵字了，但我認為還是寫一下會比較好 -->
  &#x3C;title>HTML CheatSheet&#x3C;/title>
  &#x3C;meta name="description" content="A brief page description" />
  &#x3C;meta name="keywords" content="html,cheatsheet" />
  &#x3C;!--  -->

  &#x3C;!-- 這些是跟SEO比較直接的設定、除了能夠讓機器人爬蟲了解資訊外，當你分享網站時，若有跳出分享的網址縮圖，也會顯示出設定的標題、描述跟圖片 -->
  &#x3C;meta property="fb:admins" content="YourFacebookUsername" />
  &#x3C;meta property="og:title" content="HTML CheatSheet" />
  &#x3C;meta property="og:type" content="website" />
  &#x3C;meta property="og:url" content="https://htmlcheatsheet.com/" />
  &#x3C;meta
    property="og:image"
    content="https://htmlcheatsheet.com/images/html-cheatsheet.jpg"
  />
  &#x3C;meta property="og:description" content="A brief page description" />

  &#x3C;!-- 是設定若你在Apple想將這個網站新增到桌面時顯示的App Icon -->
  &#x3C;link rel="apple-touch-icon" href="apple-touch-icon.png" />

  &#x3C;!-- 跟多語系相關的設定、指的是這個網站的原始版本的其他替代版本，然後hreflang是語系 -->
  &#x3C;link
    rel="alternate"
    hreflang="es"
    href="https://htmlcheatsheet.com/spanish/"
  />

  &#x3C;!-- import .css檔 -->
  &#x3C;link rel="stylesheet" href="/styles.css" />

  &#x3C;!-- import js 檔 -->
  &#x3C;script src="/script.js">&#x3C;/script>
&#x3C;/head>
</code></pre>
<p>Head Tags 是不會出現於網頁視覺畫面上的 tags，它的功用主要是這個網頁的設定、SEO 資訊。</p>

<h3 id="htaccess">.htaccess</h3>

<h3 id="box-model">Box Model</h3>
<p>第一個重要的概念就是 Intrinsic size (內部尺寸) vs Extrinsic sizing (外部尺寸)。</p>
<p>我記得這一路來都是一邊實作一邊隨便抄人家的 CSS 來改，早就實作過這兩種概念，但卻不知道它們名稱。</p>
<p>| Intrinsic size (內部尺寸)        | Extrinsic sizing (外部尺寸)                                                                                                                 |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| 主要是根據元素的內容決定尺寸大小 | 透過 CSS 屬性（例如 width、height、min-width、max-width、min-height 和 max-height）來設置，無論元素的內容多大多寬，外部尺寸就是固定的尺寸。 |</p>
<p><strong>值得注意的是，intrinsic sizing (內部尺寸) 是瀏覽器的 default 設定，內部尺寸通常比外部尺寸來的更彈性。</strong></p>
<p>我個人在刻 CSS 時也比較喜歡採用內部尺寸，除非是一些需要固定長寬以及考量到響應式網站的內容排版，才會去使用外部尺寸，內部尺寸使用好的話，排版跟響應式的縮放是很舒適的。</p>
<p>以下是幾個跟尺寸相關的屬性</p>
<p><img src="https://mingjungyu.files.wordpress.com/2024/05/screenshot-2024-05-15-at-9.55.31e280afpm.png" alt=""></p>
<ul>
<li>padding: div 的邊界落在 border box 的部分，padding 就是 div 和內容中間要隔多少空格。值得注意的是如果你有設<a href="https://tailwindcss.com/docs/overflow">overflow</a>屬性，那那條 scroll bar 會落在 padding 區域。</li>
<li>margin: 是圍繞在 div 外的一圈邊界，一些屬性像是 outline 跟 box-shadow 會佔據 margin 的空間。</li>
</ul>
<p>通常 <code>&#x3C;div></code> 的 default 值是 <code>display:block</code>，但其實 display 有很多不同的 value: 參照<a href="https://tailwindcss.com/docs/display">TailwindCSS display</a></p>
<p>而且，每個 HTML element 的 display 都不同，<code>&#x3C;div></code>的 default 是<code>block</code>，<code>&#x3C;li></code>的 default 是<code>list-item</code>，<code>&#x3C;span></code>是<code>inline</code>。</p>
<p>再來是 box-sizing: border-box; vs box-sizing: content-box;</p>
<p>參照 <a href="https://tailwindcss.com/docs/box-sizing">TailwindCSS</a></p>
<pre><code class="language-css">.my-box {
  width: 200px;
  border: 10px solid;
  padding: 20px;
}
</code></pre>
<pre><code class="language-css">.my-box {
  box-sizing: border-box;
  width: 200px;
  border: 10px solid;
  padding: 20px;
}
</code></pre>
<p>CSS 預設使用 default box-sizing: content-box，所以第一個 css 實際上寬度是 260px，而第二個是 200px (總寬)。</p>
<h3 id="selector">Selector</h3>
<h4 id="selector-type">Selector Type</h4>
<p>再來進入 Selector 的概念，基本上本人也只用過幾個，最常用的還是 <code>.class</code>，正所謂 .class 從頭到尾。</p>
<ul>
<li>Universal selector (*): also known as a wildcard—matches any element.</li>
<li>Type selector (HTML element): matches a HTML element directly. (就是針對某個 HTML Tag 賦予機關槍式的 style，使某個 HTML Tag 都是同個 style)</li>
<li>Class selector (.class): matches any element that has that class applied to it. (A HTML element can have one or more items defined in their class attribute) - 個人很常使用這項，基本上就是寫在 <code>&#x3C;div class=""></code>，而且一個 class 用空格分開可以寫好多個 class。</li>
<li>ID selector (#id): 通常拿來綁跟 JS 有關 (我說我)</li>
<li>Attribute selector ([data-type='primary'] or [data-type] without specific value): elements that have a certain HTML attribute, or have a certain value for an HTML attribute (長知識了，很少用)
<ul>
<li>You can use case-sensitive attribute selectors by adding an s operator to your attribute selector.</li>
<li>by adding an s: [data-type='primary' s]</li>
<li>This means that if a HTML element had a data-type of Primary, instead of primary, it would not get red text. You can do the opposite—case insensitivity—by using an i operator.</li>
</ul>
</li>
<li>Grounping selectors: A selector doesn't have to match only a single element. You can group multiple selectors by separating them with commas</li>
<li>Complex selectors</li>
<li>Compound selectors</li>
</ul>
<p><strong>Note: Attribute selector</strong></p>

[href*="example.com"] {

[href^="https"] {
  color: green;
}

/* A href that ends with .com */
[href$=".com"] {
  color: blue;
