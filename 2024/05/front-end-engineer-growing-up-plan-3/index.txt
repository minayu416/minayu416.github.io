<h2 id="css">CSS</h2>
<h3 id="the-cascade">The Cascade</h3>
<p>The cascade 是用來解決當許多 css 規則造成矛盾跟衝突時，哪些 css 樣式跟規則會優先被史用。</p>
<ul>
<li>Position and order of appearance: the order of which your CSS rules appear (你擺放 css 的位置)</li>
<li>Specificity: an algorithm which determines which CSS selector has the strongest match (不同的 selector 優先度不同，這個在下個章節有更詳細的分數計算)</li>
<li>Origin: the order of when CSS appears and where it comes from, whether that is a browser style, CSS from a browser extension, or your authored CSS (是瀏覽器本身、使用者套用、CSS Framework 的優先度)</li>
<li>Importance: some CSS rules are weighted more heavily than others, especially with the !important rule type (!important 就是非常重要，通常優先度很高、但是 transition 的優先度會比!important 高)</li>
</ul>
<h4 id="position-and-order-of-appearance">Position and order of appearance</h4>
<p>後者宣告的永遠贏，就算是 link sheet (<code>&#x3C;link rel="stylesheet" href="./test.css" /></code>) 和 <code>&#x3C;style></code> tag，混在一起時，也是自然而然最後宣告的 css 樣式優先使用。</p>
<p>但然而，如果最後宣告的樣式該瀏覽器不支援，那就會渲染倒數第二個宣告的樣式，以此類推。</p>
<h4 id="specificity">Specificity</h4>
<pre><code class="language-html">&#x3C;h1 class="my-element">Heading&#x3C;/h1>
</code></pre>
<pre><code class="language-css">.my-element {
  color: red;
}

h1 {
  color: blue;
}
</code></pre>
<p>以上的舉例來說， <code>class</code> 優先度高於 <code>html tag</code>。</p>
<p>然後，id 的優先度會高於所有其他元素，所以建議不要直接用 id 撰寫 css 樣式，如果有其他需要 override 的需求的話，會比較難維護。</p>
<p><code>a.my-class.another-class[href]:hover</code>，雖然可以用這種方式對 css 命名指定(selector)，但會造成維運困難或者若需要樣式可重複利用的話，建議讓 selector 越簡單越好。</p>
<p>這個部分的分數計算在下個章節會有更詳細的解說。</p>
<h4 id="origin">Origin</h4>
<p>The order of specificity of these origins, from least specific, to most specific is as follows:</p>
<p>因為會有來自不同地方 import 的 css 樣式，包括瀏覽器原生有的，其優先度如下：</p>
<p>低 -> 高</p>
<ul>
<li>User agent base styles. These are the styles that your browser applies to HTML elements by default.</li>
<li>Local user styles. These can come from the operating system level, such as a base font size, or a preference of reduced motion. They can also come from browser extensions, such as a browser extension that allows a user to write their own custom CSS for a webpage.</li>
<li>Authored CSS. The CSS that you author.</li>
<li>Authored !important. Any !important that you add to your authored declarations.</li>
<li>Local user styles !important. Any !important that come from the operating system level, or browser extension level CSS.</li>
<li>User agent !important. Any !important that are defined in the default CSS, provided by the browser.</li>
</ul>
<p>!important -> 不是不重要，反而是最重要，優先度最高。</p>
<h4 id="importance">Importance</h4>
<p>Not all CSS rules are calculated the same as each other, or given the same specificity as each other.</p>
<p>The order of importance, from least important, to most important is as follows:</p>
<ul>
<li>normal rule type, such as font-size, background or color</li>
<li>animation rule type</li>
<li>!important rule type (following the same order as origin)</li>
<li>transition rule type</li>
</ul>
<p>animation 跟 transition 的優先度會比較高，主要是因為當有這兩個樣式時，他們主要是改變視覺狀態的因子。</p>
<p><strong>Note: 最後就是善用開發者工具去觀察 css 樣式的優先層級。</strong></p>
<p>腦子是不用這麼可憐塞這些計算，開發者工具是你的好朋友。</p>
<h3 id="specificity-1">Specificity</h3>
<p>這個章節主要是針對上一個章節 The Cascade 的 Specificity 去做補充。</p>
<p>Specificity 對於每個 selector 都有一個分數制，哪一個 selector 擁有高分，他的優先度最高被採用作為渲染 css 樣式，打個比方:</p>
<pre><code class="language-html">&#x3C;button class="branding">Hello, Specificity!&#x3C;/button>
</code></pre>
<pre><code class="language-css">button {
  color: red;
}

.branding {
  color: blue;
}
</code></pre>
<p>按鈕會是藍色的，因為就 <code>HTML tag</code> 跟 <code>class selector</code>，class 的分數會高一些。</p>
<h4 id="specificity-scoring">Specificity scoring</h4>
<p>以下列舉不同的 selector 以及他們的分數</p>
<ul>
<li>Universal selector (*): no specificity and gets 0 points</li>
<li>Element or pseudo-element selector (div, ::selection): 1 point of specificity.</li>
<li>Class, pseudo-class, or attribute selector (.my-class, :hover, [href='#']): 10 points of specificity.</li>
<li>ID selector (#id): 100 分</li>
<li>Inline style attribute (<code>&#x3C;div style="color: red">&#x3C;/div></code>): 1000 分</li>
<li>!important rule: 10000 分</li>
</ul>
<p><strong>Note: pseudo-class 例外，當使用 :not() 等同使用 pseudo-class，所以下面這個分數是 11 分，div 1 分 + pseudo-class 10 分</strong></p>
<pre><code class="language-css">div:not(.my-class) {
  color: red;
}
</code></pre>
<p><strong>Note: !important</strong></p>
<pre><code class="language-css">.my-class {
  color: red !important; /* 10,000 points */
  background: white; /* 10 points */
}
</code></pre>
<p>簡單來說可以累加</p>
<pre><code class="language-html">&#x3C;a class="my-class another-class" href="#">A link&#x3C;/a>
</code></pre>
<pre><code class="language-css">/* 1分 */
a {
  color: red;
}

/* 11分 */
a.my-class {
  color: green;
}

/* 21分 */
a.my-class.another-class {
  color: rebeccapurple;
}

/* 31分 */
a.my-class.another-class[href] {
  color: goldenrod;
}

/* 41分 */
a.my-class.another-class[href]:hover {
  color: lightgrey;
}
</code></pre>
<h4 id="visualizing-specificity">Visualizing specificity</h4>
<p>這個部分參考了 chatGPT 的回答才會計算，簡單來說上半部已經介紹了如何計算分數，這個章節比較像是當所有的 selector 都拼在一起時，你可以怎麼用各種進位制計算。</p>
<p>chatGPT 的回答是 specificity 分數由四個部分組成數字(文章是 3 個)</p>
<ol>
<li>Inline styles (1000 分)</li>
<li>ID (100 分)</li>
<li>Class, pseudo-class, or attribute selector (.my-class, :hover, [href='#']) (10 分)</li>
<li>Element or pseudo-element selector (div, ::selection) (0 分)</li>
</ol>
<p>然後以上的數字用進位制去計算</p>
<p>Inline styles - ID - Class - Element</p>
<pre><code class="language-css">/* Example 1 */
div {
  color: blue;
}
</code></pre>
<ol>
<li>Inline styles: 沒有</li>
<li>ID: 沒有</li>
<li>Class, pseudo-class, or attribute selector (.my-class, :hover, [href='#']): 沒有</li>
<li>Element or pseudo-element selector (div, ::selection): 1 個</li>
</ol>
<p>所以分數是 0-0-0-1，就是 1 分。</p>
<pre><code class="language-css">/* Example 2 */
ul li a:hover {
  color: purple;
}
</code></pre>
<ol>
<li>Inline styles: 沒有</li>
<li>ID: 沒有</li>
<li>Class, pseudo-class, or attribute selector (.my-class, :hover, [href='#']): 1 個 (hover)</li>
<li>Element or pseudo-element selector (div, ::selection): 3 個 (ul, li, a)</li>
</ol>
<p>分數是 0-0-1-3，就是 13 分</p>
<p>來看看網站內的範例</p>
<pre><code class="language-css">a.my-class.another-class[href]:hover {
  color: lightgrey;
}
</code></pre>
<ol>
<li>Inline styles: 沒有</li>
<li>ID: 沒有</li>
<li>Class, pseudo-class, or attribute selector (.my-class, :hover, [href='#']): 4 個 (.my-class, .another-class[href], :hover)</li>
<li>Element or pseudo-element selector (div, ::selection): 1 個 (a)</li>
</ol>
<p>所以分數是 0-0-4-1 (網站寫 0-4-1，因為他沒有把 Inline styles 算進去)， 41 分</p>
<p>另一個網站範例</p>
<p>Which of the following selectors is 1-2-1?</p>
<pre><code class="language-css">#specialty: hover li.dark;
</code></pre>
<ol>
<li>Inline styles: 沒有</li>
<li>ID: 1 個 (#)</li>
<li>Class, pseudo-class, or attribute selector (.my-class, :hover, [href='#']): 2 個 (:hover, .dark)</li>
<li>Element or pseudo-element selector (div, ::selection): 1 個 (li)</li>
</ol>
<p>所以答案是 0-1-2-1 (1-2-1), 121 分</p>
<h4 id="pragmatically-increasing-specificity">Pragmatically increasing specificity</h4>
<pre><code class="language-css">.my-button {
  background: blue;
}

button[onclick] {
  background: grey;
}
</code></pre>
<pre><code class="language-html">&#x3C;button class="my-button" onclick="alert('hello')">Click me&#x3C;/button>
</code></pre>
<p>接著以上的答案來計算的話</p>
<ul>
<li>.my-button: 10 分</li>
<li>button[onclick]: 11 分，因為 button 是 1 分、[onclick]算 attribute selector 所以 10 分</li>
</ul>
<pre><code class="language-css">.my-button {
  background: blue;
}

[onclick] {
  background: grey;
}
</code></pre>
<p>這樣的話兩週都是 10 分，可是 grey 贏，因為他是在最後被宣告。</p>
<p>所以如果想要按鈕是 blue 的話，倒過來寫就可以了，分數相同的，以最後宣告的為主。</p>
<pre><code class="language-css">[onclick] {
  background: grey;
}

.my-button {
  background: blue;
}
</code></pre>
<p>這樣按鈕就是 blue。</p>
<h3 id="inheritance">Inheritance</h3>
<p>後來發現這個章節跟前面相關，所以就將 Inheritance 更新在此篇文章。</p>
<p>先來引用這個章節的第一個例子，但我做了些改變</p>
<pre><code class="language-css">.my-button {
  display: inline-block;
  padding: 1rem 2rem;
  text-decoration: none;
  background: pink;
  font: inherit;
  text-align: center;
  color: blue;
}

article a {
  color: maroon;
}
</code></pre>
<pre><code class="language-html">&#x3C;a href="http://example.com" class="my-button">I am a button link&#x3C;/a>
</code></pre>
<p>你認為 a link 的字的顏色是什麼顏色？</p>
<p>如果結合上面兩個章節來看，會是 blue。因為在前兩個章節提到， class 的 selector 分數比 html attribute (article, a)高，所以在優先度來看，會優先選擇 <code>class=".my-button"</code>的樣式，再選擇 <code>article a</code>，如果他內部有其他樣式是<code>.my-button</code>沒有定義的。</p>
<h4 id="how-to-explicitly-inherit-and-control-inheritance">How to explicitly inherit and control inheritance</h4>
<p>接著本章重點介紹這三個相關的關鍵字怎麼使用</p>
<ul>
<li>The inherit keyword</li>
<li>The initial keyword</li>
<li>The unset keyword</li>
</ul>
<h5 id="inherit">Inherit</h5>
<p><code>inherit</code> 關鍵字就是字面上的意思「繼承」，使用後他會繼承父元素所設定的值。</p>
<pre><code class="language-css">.parent {
  color: red;
}
.child {
  color: inherit; /* child 會繼承 parent 設定的值，也就是 color: red */
}
</code></pre>
<p>但這例子很抽象，再舉一個</p>
<pre><code class="language-css">strong {
  font-weight: 900;
}

.my-component {
  font-weight: 500;
}

.my-component strong {
  font-weight: inherit;
}
</code></pre>
<pre><code class="language-html">&#x3C;div class="my-component">
  &#x3C;strong>test &#x3C;/strong>
&#x3C;/div>
</code></pre>
<p>今天想要特別在<code>class="my-component"</code>底下的<code>strong</code>他的 font-weight 繼承<code>class my-component</code> (即 500)，我就用上面列舉的寫法</p>
<pre><code class="language-css">.my-component strong {
  font-weight: inherit;
}
</code></pre>
<p>這樣，所有在 <code>div class="my-component"</code>底下的<code>strong</code>，他們的 font-weight 就都會是 500，繼承<code>class my-component</code>所設定。</p>
<h5 id="initial">Initial</h5>
<p>而<code>initial</code>這個關鍵字，則是賦予你有重置(reset)的功能，然後再將值繼承最<code>default</code>的值(通常是瀏覽器原生的值)。</p>
<pre><code class="language-css">aside strong {
  font-weight: initial;
}
</code></pre>
<p>比方說，他把原先 <code>strong</code>的設定透過 <code>initial</code> 重設， 那原先 strong 會是<code>粗體字</code>或者設為 <code>font-weight: 900;</code>，重置後，字會變成瀏覽器原先設計的粗細 (也許是<code>細字</code>或<code>font-weight: 200;</code>)。</p>
<h5 id="unset">Unset</h5>
<p>最後則是 <code>unset</code> 關鍵字，這個關鍵字有兩種用法，「他<code>保留</code> 擁有繼承屬性的值、但<code>重設</code> 沒有繼承屬性的值」。</p>
<p>舉下面這個例:</p>
<pre><code class="language-css">.foo {
  color: blue;
}
.bar {
  color: green;
}
p {
  color: red;
}
.bar p {
  color: unset;
}
</code></pre>
<p>在這樣的情況下 <code>.bar p</code> 的字會是綠色，因為 <code>&#x3C;div class=".bar"></code> 是父元素，底下的 <code>&#x3C;p></code> 繼承父元素的顏色，所以是綠色。</p>
<pre><code class="language-css">div {
  border: 1px solid green;
}
p {
  border: 1px solid red;
}
.bar p {
  border-color: unset;
}
</code></pre>
<p>在這個例子中，<code>&#x3C;p></code>設置的邊框樣式是 <code>1px solid red;</code>，邊框顏色是紅色。但是在 <code>.bar p</code> 樣式時，<code>unset</code> 了 <code>border-color</code> 這個值，我們可以看到上面的 selector 都沒有賦予 <code>border-color</code>這個屬性，所以在這 <code>.bar p</code> 邊框顏色其實會被重設為跟瀏覽器一樣 (通常是黑色)。</p>
<h3 id="本節小課程---css-大小單位">本節小課程 - CSS 大小單位</h3>
