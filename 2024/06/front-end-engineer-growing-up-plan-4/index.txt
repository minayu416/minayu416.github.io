<h2>CSS</h2>
<h3>Color</h3>
<p>探討過 css 樣式的優先度計算後，此篇當進到色彩與尺寸大小，和 css 樣式屬性較爲相關連的內容。</p>
<h4>Where to use color in CSS rules</h4>
<p>在網站內容的最下方，但我決定將它移動到上方先解釋。</p>
<p>只要是 屬性的值 <code>data-type</code> 為 <code>color</code> 的值欄位都可以放 color 值，所以相關的屬性:</p>
<ul>
<li>color: 文字</li>
<li>text-shadow</li>
<li>text-decoration-color</li>
<li>border</li>
<li>border-color</li>
<li>background-color 或 background: 背景顏色</li>
</ul>
<p>都可以，然後 color 也可以使用各種漸層效果，像是 <code>linear-gradient</code>，這個會在之後的篇幅特別有一篇介紹。</p>
<p>那麼定義或形成顏色的值主要兩大項幾個小項目</p>
<ul>
<li>Numeric colors (由數字、英文字組成)
<ul>
<li>Hex colors</li>
<li>RGB (Red, Green, Blue)</li>
<li>HSL (Hue, Saturation, Lightness)</li>
<li>...</li>
</ul>
</li>
<li>Color Keywords (該英文單字對應一個固定的顏色)</li>
</ul>
<h4>Numeric colors</h4>
<p>在此項目內容中，有介紹三種模式組成顏色，分別是以下</p>
<ul>
<li>Hex colors</li>
<li>RGB (Red, Green, Blue)</li>
<li>HSL (Hue, Saturation, Lightness)</li>
</ul>
<p>但我後來去翻文獻時，我發現還蠻多調整顏色的模式。</p>
<p>詳細可以看<a href="https://ithelp.ithome.com.tw/articles/10340417">這篇文章</a></p>
<p>像本人有接觸到彩妝，就有接觸過 HSL，以及與設計師好朋友的聊天內容，也接觸過 <code>CMYK</code>，不過還有很多種不同調顏色的模式，所以詳細狀況，還是要看需求而定。</p>
<p>今天先針對三種介紹。</p>
<p><strong>Hex colors</strong></p>
<p>通常由 <code>6位數</code> 或者 <code>8位數</code> 的 0-9 跟 A-F 數值組成，他們會被翻譯成 RGB 原始三元素。</p>
<ul>
<li><code>6位字</code>組成一個顏色，例如: #b71540、#60a3bc、#f6b93b</li>
<li><code>8位字</code>也就是原本<code>6位字</code>的顏色加上額外<code>2位</code>可以顯示他的<code>透明度</code>，例如: #00000080 (黑色, 透明 50%)、#000000BF(黑色, 透明 75%)</li>
</ul>
<p>網路上都蠻多 Hex colors 網站可以慢慢選，我倒是覺得不用特別去記怎麼計算。</p>
<p><strong>RGB (Red, Green, Blue)</strong></p>
<p>怎麼可以缺少色彩學大神 RGB 三元素呢。</p>
<p>通常會用函式 <code>rgb()</code>包裹，內含三個值(紅綠藍) <code>0~255</code> 或 <code>百分比%</code>顯示，例如</p>
<ul>
<li>rgb(255, 255, 255) or rgb(100%, 100%, 100%).</li>
<li>rgb(113, 221, 101) or (%數我不知道 XD).</li>
</ul>
<p>那如果要調透明度的話，有兩種方式</p>
<ul>
<li>保持使用 <code>rgb()</code>: 需要新增 / 在函式內，比方說: rgb(0 0 0 / 50%) or rgb(0 0 0 / 0.5)</li>
<li>改用 <code>rgba()</code>: rgba(0, 0, 0, 50%) or rgba(0, 0, 0, 0.5)</li>
</ul>
<p>兩者都能增加透明度。</p>
<p><strong>HSL (Hue, Saturation, Lightness)</strong></p>
<pre><code class="language-css">h1 {
  color: hsl(344, 79%, 40%);
}
</code></pre>
<p>HSL 代表<code>色調</code>、<code>飽和度</code>跟<code>明亮度</code>，通常我比較常在彩妝或者服飾穿搭上看到相關的名詞，css 上很少遇到這個案例。</p>
<p>使用 hsl() 函式包裹</p>
<ul>
<li>Hue (色調) - 值為 0-360 度</li>
<li>Saturation (飽和度) - 0% ~ 100%</li>
<li>Lightnextt (明亮度) - 0% ~ 100%</li>
</ul>
<p>那一樣如果要定義透明度 alpha，跟 rgb 很相似，兩種方式:</p>
<ul>
<li>同樣使用 hsl() - hsl(0 0% 0% / 50%) or hsl(0 0% 0% / 0.5)</li>
<li>改用 hsla() - hsla(0, 0%, 0%, 50%) or hsla(0, 0%, 0%, 0.5)</li>
</ul>
<h4>Color Keywords</h4>
<p>再來是有命名的顏色，目前看文章是寫有 148 種顏色在 css 是有直接用英文命名的。</p>
<p>有興趣在自己看網站列出的<a href="https://web.dev/learn/css/color#color_keywords">顏色命名表</a>。</p>
<p>最後特別介紹兩個單詞</p>
<ul>
<li>transparent: 這個是代表一個完全透明的顏色</li>
<li>currentColor: 這個詞指的是現在的顏色，比方說如果 color 設 red，那你 border-color 填 currentColor，則 border-color 也會是 red。</li>
</ul>
<h3>Sizing Units</h3>
<p>Sizing Units 這個章節介紹的內容跟上一篇文章的小課程內容相關，主要是介紹各個 css 屬性值的不同尺寸單位。</p>
<p>那麼因應 CSS 跟響應式網站的發展，前端網頁設計的技術會越來越精緻，從原本常見的 <code>px</code> 去調整各個空間、文字的絕對大小，到後來延伸出 <code>rem</code>, <code>%</code> 之類的單位依據 <code>子元素</code> 或 <code>default數值</code> 為基準放大縮小因應響應式設計，究竟有多少不同的單位呢？會在此篇介紹。</p>
<ul>
<li>ch</li>
<li>Number</li>
<li>Percentage</li>
<li>Dimensions and lengths
<ul>
<li>Absolute lengths</li>
<li>Relative lengths</li>
<li>Viewport-relative units (?)</li>
</ul>
</li>
<li>Miscellaneous units
<ul>
<li>Angle units</li>
<li>Resolution units (?)</li>
</ul>
</li>
</ul>
<h4>ch</h4>
<p>雖不明白為什麼開場用 ch 這個單位，但確實好像也不知道這單位該屬於哪個分類，那就從 <code>ch</code> 開始講起吧。</p>
<ul>
<li>ch 這個單位是指等同一個字元 <code>0</code> 的 <code>寬度</code>大小，你可以使用 <code>ch</code> 來度量 <code>width</code>，假設 <code>70ch</code> 則為等同 70 個 <code>0</code> 字元的寬度。</li>
</ul>
<p>範例:</p>
<pre><code class="language-css">h1 {
  font-size: 3.5rem;
  line-height: 1.2;
  max-width: 15ch;
}

p {
  font-size: 1.25rem;
  max-width: 70ch;
}
</code></pre>
<p>這特別運用在某些文章需要固定寬高的情況下，如果想用的是文字的寬度為尺寸，比方說寬度限定 40 個英文字，而不是固定的 <code>px</code> 或 <code>cm</code> 這種尺寸時的場景。</p>
<h4>Number</h4>
<p>基本上，大部分的 css 屬性的值都是使用數字作為單位，像是 opacity(透明度)、line-height(行高)、rgb(color 色彩)，有的是用整數(1,2,3...,100)有的是小數(0.1, 0.2,...)</p>
<p>範例</p>
<pre><code class="language-css">p {
  font-size: 24px;
  line-height: 1.5;
}
</code></pre>
<p>其中 1.5 這個小數指的是放大 150% 倍的 <code>&#x3C;p></code>尺寸，所以假設原本字的大小是 <code>24px</code>放大後變成<code>36px</code>。</p>
<p>其他案例</p>
<ul>
<li><code>filter: sepia(0.5)</code> -> 50% sepia filter to the element.</li>
<li><code>opacity: 0.5</code> -> 50% 透明度</li>
<li><code>rgb(50, 50, 50)</code> -> 各 50 的 R, G, B 色彩 (0-255)</li>
<li><code>transform: scale(1.2)</code> 放大元素 120% 大小</li>
</ul>
<h4>Percentage</h4>
<p>除了用數字去精準訂定尺寸之外，還可以使用百分比作為尺寸單位。</p>
<pre><code class="language-css">div {
  width: 300px;
  height: 100px;
}

div p {
  width: 50%;
}
</code></pre>
<p>假設 div 長跟寬 分別是 100px 跟 300px，那 div 下的文字段 p 就會是寬度*50% -> 150px 寬。</p>
<p>除了長寬以外，<code>transform: translateX</code>、剛剛列舉的透明度、跟其他元素也能使用百分比作為單位。</p>
<p>但需要注意的是 (我還沒查證過 XD)，使用百分比作為單位時%，要注意子元素或者初始值(default)有沒有先設原先的數值，如果沒有那就算寫多少百分比，也會沒有效用。</p>
<h4>Dimensions and lengths</h4>
<p>再來就是更多單位的介紹，分為</p>
<ul>
<li><a href="https://web.dev/learn/css/sizing#absolute_lengths">Absolute lengths (絕對)</a></li>
<li><a href="https://web.dev/learn/css/sizing#relative_lengths">Relative lengths (相對)</a></li>
</ul>
<p>更詳細的介紹可以直接點網站查看。</p>
<p><strong>Absolute lengths (絕對長度)</strong> 包括像是常用得 <code>px</code>，還有 <code>cm</code>、<code>mm</code>...等許多不同的尺寸標示具體的固定的大小，比方說<code>cm</code>就是標記具體的幾公分，<code>px</code>也是早期蠻常被使用做固定長寬大小的單位。</p>
<p>但隨著響應式網站的需求增多，很多新設計的網站比較少使用這類型的絕對長度來訂定尺寸，因為當網頁縮放不同的裝置 (大螢幕的網站、中螢幕的網站、不同尺寸的平板及不同尺寸的手機)時，這類型寫固定(寫死)的單位就變得很不靈活。</p>
<p>因此，現在大部分都使用相對長度的單位來訂定尺寸。</p>
<p><strong>Relative lengths (相對長度)</strong></p>
<ul>
<li>Font-size-relative units (字體大小相對單元)</li>
<li>Viewport-relative units (視圖大小相對單元)</li>
</ul>
<p>(中文真不知道怎麼翻比較好)</p>
<p>相對長度跟絕對長度不同，絕對長度通常是寫固定的大小，比方說<code>3cm</code> 或者 <code>20px</code>(Pixels)，而相對長度大多是以比方說<code>一個單字的大小</code>或者<code>父元素的基準為縮放</code>。</p>
<p>假設以<code>rem</code>舉例，如果根元素(root element)的字大小(font size) 為 <code>16px</code>，那麼 <code>1rem</code> = <code>16px</code>，<code>2rem</code> = <code>32px</code> 以此類推。</p>
<p>相對長度(Relative lengths)又分為<code>Font-size</code>跟<code>Viewport</code>兩種相對單元。</p>
<p><strong>1. Font-size-relative units</strong></p>
<p>開頭的 <code>ch</code> 就是一種相對長度，因為他並沒有寫死固定為幾公分，而是透過一個文字的寬度做定義，所以<code>1ch</code>等同一個字的寬度，而<code>2ch</code>等同兩個字的寬度。</p>
<p>我本人比較常看到的是 <code>rem</code> 跟 <code>em</code>，尤其現在熱門的 css 框架 <code>TailwindCSS</code> 就是使用 <code>rem</code> 作為單位，而這類型的相對尺寸是基於根元素(root element)的初始值去縮放其絕對長度。rem 的計算上面有提及了就不再寫一次啦。</p>
<p>更多的 Font-size-relative 單位可以查看<a href="https://web.dev/learn/css/sizing#font-size-relative_units">web.dev 網站內容</a></p>
<p><strong>2. Viewport-relative units</strong></p>
<p>這個單元我比較不熟，比較少看到。</p>
<p>解釋的話是運用瀏覽器視窗的尺寸作為相對基礎來縮放大小，我想可以理解為直接用瀏覽器的長寬拆解成單位，依照比例去縮放吧。</p>
<p>印象比較深刻是我先前在用<code>react.js</code>跟<code>next.js</code> 開發網站時，當時我希望我的網頁背景是固定滿版，但我不管是設定了</p>
<pre><code class="language-css">#root {
  height: 100%;
}
</code></pre>
<p>或其他 css 都沒有用處，後來我去查文件發現若需要固定滿版不能滾動的話，高度需要這樣設定：</p>
<pre><code class="language-css">#root {
  height: 100%;
  height: 100vh;
  height: 100dvh;
}
</code></pre>
<p><code>vh</code>: 瀏覽器尺寸 1%的高度，所以<code>100vh</code>就是<code>瀏覽器100%</code>高度，也就是讓背景跟網站固定滿版呈現。</p>
<p>這大概是我第一次遇到這種 Viewport-relative 的單位，由於遇到的次數不多，所以不知道未來會不會常使用，或者說未來若遇到同樣的問題，就能用這類型的單位解決。</p>
<p>更多的 Viewport-relative 單位可以查看<a href="https://web.dev/learn/css/sizing#viewport-relative_units">web.dev 網站內容</a></p>
<h4>Miscellaneous units</h4>
<p>最後一章節是關於其他單位的介紹。</p>
<p>首先是角度 (Angle units)，當你需要旋轉某個物件或者旋轉效果，其單位是使用 deg，旋轉<code>360度</code>就是<code>360deg</code>。</p>
<pre><code class="language-css">div {
  width: 150px;
  height: 150px;
  transform: rotate(60deg);
}
</code></pre>
<p>有其他的角度單位分別是 rad (radians), grad (gradians) 和 turn， 1turn = 360deg, 0.5turn = 180deg.</p>
<p>再來是解析度 (Resolution units) 的單位 (dpi)，比方說 192 解析度就是 <code>192dpi</code>。</p>
