<h3 id="1-what-is-http">1. What is HTTP?</h3>
<p>HTTP stands for HyperText Transfer Protocol. It is an application protocol that allows for transfer data. HTTP enables the retrieval of resources like HTML pages, image, videos, from servers to browsers.</p>
<p>It follows a client-server model, where a client (such as a web browser) sends requests to a server, and the server responds with the requested resources. HTTP uses various methods to perform different actions on resources.</p>
<h4 id="extend-延伸題-osi-七層模型">Extend 延伸題: OSI 七層模型</h4>
<ul>
<li>實體層（physical layer）</li>
<li>資料連結層（data link layer）</li>
<li>網路層（network layer）</li>
<li>傳輸層（transport layer）</li>
<li>會議層（session layer）</li>
<li>表現層（presentation layer）</li>
<li>應用層（application layer）</li>
</ul>
<h4 id="extend-延伸題-可否列舉其他-protocol-請問某某-protocol-的作用">Extend 延伸題: 可否列舉其他 Protocol? 請問某某 Protocol 的作用？</h4>
<h4 id="extend-延伸題-what-is-three-way-handshake-三次握手-and-four-way-handshake-四次握手">Extend 延伸題: What is three-way handshake (三次握手) and four-way handshake (四次握手)?</h4>
<p>三次握手和四次握手(揮手)是 TCP 建立與斷開連線的過程。</p>
<p>以理論上來說，TCP 協議是確保在兩個端點之間建立和維護可靠的數據傳輸，所以連線一但通過建立後，其連線就會一直保持，直到斷開。</p>
<ul>
<li>三次握手: 為 TCP 協議建立連線的過程。</li>
<li>四次握手(揮手): 為 TCP 協議斷開連線的過程。</li>
</ul>
<p>若以實作上來說，運用在 TCP 協議上有眾多協議，舉例的話有 HTTP 及 Websocket 在前端與後端的實作上較為廣泛使用。只是，HTTP 主要基於 Client-Server Model，也就是由客戶端傳需求至伺服器端，再由伺服器端收到需求後處理完回傳回應(Response)，每次 HTTP 需求都是獨立且斷開，而且只能由客戶端發送單向請求。Websockets 則是適合資料即時串流以及雙向資料傳輸，建立在 TCP 協議的穩定連線上，使資料能夠雙向交流，且確保資料能夠穩定傳輸。</p>
<h4 id="extend-延伸題-那-udp-是什麼其特性又是什麼">Extend 延伸題: 那 UDP 是什麼？其特性又是什麼？</h4>
<h3 id="2-what-does-an-http-request-consist-of">2. What does an HTTP request consist of?</h3>
<p>Each HTTP request consists of three parts, which are transmitted in the specified order:</p>
<ol>
<li>Starting line (Request line): defines the type of the message.</li>
</ol>
<pre><code>GET /api/index.html HTTP/1.1
</code></pre>
<ul>
<li>the method is GET</li>
<li>the path is /api/index.html</li>
<li>the HTTP version is HTTP/1.1</li>
</ul>
<ol start="2">
<li>Headers: characterize the message body, transmission parameters, and other information.</li>
</ol>
<pre><code>Accept-Language: fr, de
If-Modified-Since: Fri, 10 Dec 2004 11:22:13 GMT
</code></pre>
<ol start="3">
<li>Message body: contains the actual data of the message. It must be separated from the headers by an empty line.</li>
</ol>

<h3 id="what-are-react-props">What are React Props?</h3>
<h3 id="explain-the-concept-of-props-drilling-in-react">Explain the concept of props drilling in React.</h3>
<p>這兩題併在一起看，Props 就是跨 Components 傳遞的資料。</p>
<pre><code class="language-js">function Component(props) {
  // use Props to do something
}
</code></pre>
<p>和 python 裡在不同的 function 傳遞的函數資料是差不多的概念</p>
<pre><code class="language-py">
def function(props):
    // do something
    pass

</code></pre>
<p>差別是 React.js 的 Props 是 read-only，不可變動。</p>
<p>第二題的 props drilling 是假如今天一個父 Component 裡有好幾層子 Components，然後要把資料 Props 一直傳遞進去，這個現象就叫做 props drilling。</p>
<pre><code class="language-js">
// 這樣 props 就會層層傳遞，全家一起享用。
function Component ({props}){
    &#x3C;ChildA props={props} />
    &#x3C;ChildB props={props} />
}

function ChildA ({props}){
    &#x3C;GrandChildC props={props} />
    &#x3C;GrandChildE props={props} />
}

</code></pre>
<p>解決方式可以用 useContext Hook 去實作，但敝女目前實作的一個專案當時邊學邊做+趕，所以也是一堆 data drilling 的問題，正將用 useContext 去重構這個任務排程在未來。</p>
