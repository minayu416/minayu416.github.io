<p>首先</p>
<p>至從買了個 Mac，自己進入到一個全新的系統 Unix</p>
<p>一般使用蘋果系統的人應該比較不曉得，就是時常享受著蘋果產品帶來的質感跟炫富</p>
<p>但其實由於蘋果的系統為 Unix 系統，我個人覺得跟 Ubuntu 還蠻像的</p>
<p>但可能也是因為蘋果產品市佔率比 linux 高一點，所以有些公司會針對 ios 跟 macOS 開發軟體</p>
<p>也就是說 Mac 的系統以論上介於 Window 這樣人性化和 linux 這樣偏機器化的系統中間</p>
<p>有點偏人性化但也偏機械化</p>
<p>而自己本身是一個一半聽懂人話，一半又是理科腦想探討機器的人</p>
<p>所以對 linux 有著極大的興趣</p>
<p>以往都是學弟們將 ubuntu 架好在機器上</p>
<p>我們將寫的網路扔到 home 裡面</p>
<p>這次有幸，從 home 往前探勘，才發現許多系統的運作操作，還有很多軟體程式都裝在目錄 home 的前一層</p>
<p>也才知道 etc 是做什麼用的</p>
<p>為什麼會說 ubuntu 和 unix 很像</p>
<p>其實就是大家打開終端機，將目錄從/使用者/那端往前切後會發現</p>
<p>使用者端(應該就是 Home)之前的目錄跟資料夾，macOS 和 Ubuntu 都很像</p>
<p>應該說所有的系統操作跟軟體程式都裝在這一層</p>
<p><img src="https://minayu0416.files.wordpress.com/2018/04/screen-shot-2018-04-13-at-2-46-06-pm.png" alt="Screen Shot 2018-04-13 at 2.46.06 PM"></p>
<p>所以很多開發者的部落格跟資訊常常說將你的程式寫在根目錄</p>
<p>根目錄應該就是 Home 裡</p>
<p>因為 home 以外的其他資料夾比較偏向是系統層面的東西</p>
<p>還有如果你是新手再起初開發一些程式時，會發現有些部落客或者課程不會讓你開發的程式資料夾建在 Desktop</p>
<p>但 Windows 用久了，什麼東西除了扔桌面跟 Download 很少扔在其他地方</p>
<p>但在 Mac，既然是寫程式，其實就是將資料夾建在桌面上一層的地區</p>
<p>所以如果你想要把一些你曾經建了卻不知道建在哪的開發程式資料夾刪除</p>
<p>從桌面移動上一層目錄，應該會找到(系統應該也會幫你裝在那)</p>
<p>換句話說，在 windows 等於建在Ｃ曹的 Prgramfiles &#x3C;&#x3C; 這個東西換成中文就是程式的檔案啦～</p>
<hr>
<p>提到 Homebrew 其實是那時候 mysql 跟 python 套件那邊卡住</p>
<p>還有我卡 python2.7 跟 python3.6 的時候</p>
<p>Homebrew 是蘋果的套件管理工具</p>
<p>以前我常常把他跟 pip 搞錯</p>
<p>所以如果你裝了什麼程式或者裝什麼程式的套件會需要使用到 Homebrew</p>
<p>而因為這樣我去下了些指令找出之前用 Homebrew 亂裝了些什麼</p>
<p>早期不懂，只會看人家文章學著下指令裝</p>
<p>但可能那時候想把 python3.6 殺掉，全部只用 2.7，才想說找出 homebrew 清掉之前亂裝的東西</p>
<p>啊因為我從進研究所開始寫程式的時候接觸到的全都是 Python 居多</p>
<p>所以我以為 pip install = 全部終端機安裝的用法</p>
<p>所以拿這個指令在 ubuntu 猛按就是一直報錯 XDD</p>
<p>我應該可以這樣理解啦</p>
<p>Python 跟 pip 是綁在一起</p>
<p>Python Package Index，簡稱 PyPI</p>
<p>他是軟體包管理系統，而可以拿來用來安裝 Python 的套件</p>
<p>而不是像我因為沒有接觸過其他語言，就一直 pip pip pip xDDDD</p>
<p>然後打的很開心，報錯還一直問為什麼沒辦法裝!!! XD</p>
<hr>
<p>這解釋起來比較困難，但是要裝 phpmyadmin 就需要裝 mysql</p>
<p>然後 A 端連過去 B 端</p>
<p>B 端要裝 phpmysql</p>
<p>A 端要裝，mysql 及能連線 mysql 的套件(一樣專完 mysql 才能裝)</p>
<p>mysql 裝完記得開起來才能裝</p>
<p>mysql 跟 python 跟其他軟體都會被統一裝在一個地方(我那時候有看到可是我忘了是在哪個目錄裡)</p>
<p>最後撒花一下</p>
<p>就是學到了以往 exe 這類型的桌機檔都怎麼開發</p>
<p>其實不會很困難，我們都把它想得十分十分的困難</p>
<p>tkinter 適用 python 開發的一種介面套件</p>
<p>而開發桌機程式，我自己智障作法就是像網頁一樣先寫好幾個介面，再去一個一個定義裡面的功能，算是比較笨的方法，但我還不知道比較偏機械端的桌機版要怎麼跟資料庫連結像寫網頁那樣 XD</p>
<hr>
<p>最後說一下，成功將 bind 改後，下一個被擋的原因</p>
<p>也是我放棄改用別的的原因</p>
<p>被黨的原因是 phpmyadmin 的 database 拒絕我這個電腦 ip 的連線</p>
<p>但因為我本來就是希望開發這個程式丟給不同電腦的人使用，在將他們的資訊收集起來回送雲端</p>
<p>而為什麼會拒絕電腦 ip 的連線</p>
<p>主要就是在 phpmyadmin 裡的權限使用者那邊，需要新增</p>
<p>如果是只有你一個人要用，你就在權限使用者那新增你的電腦就好</p>
<p>但是，我是因為我要不同電腦的人回送他的資料回來</p>
<p>我總不可能手動新增每個人電腦的 ip 進來</p>
<p>再來利用像學校那樣的鎖 ip 網段，這種的可以在 phpmyadmin 上辦到，解決權限問題</p>
<p>因為鎖 ip 網段是為了安全</p>
<p>鎖權限也是為了安全</p>
<p>還有一種就是你開發註冊帳號跟密碼的系統，這樣就能達到"使用者登入"(權限開放給使用者)</p>
<p>而我的用意是希望將資料庫開發給大家回傳資料，這樣做法顯然不能用</p>
<p>Python3.6+Pymsql 的方法直接這樣搞</p>
<p>請示了一下指導老師，也是說不行</p>
<p>但得到一個新的名稱就是 REST API</p>
