9:T1888,<p>這次，我就不說自己完成幾%了</p>
<p>免得雷到自己 XD</p>
<p>至從上篇....</p>
<p>好幾週我都說我完成 80%了，但到最後都卡住卡住</p>
<p>卡住啊！</p>
<hr>
<p>至從多增加跟程式極強的學弟相處及溝通後</p>
<p>聽從他們的建議</p>
<p>我將本來我自己打算就用黑視窗呈現我的程式</p>
<p>某天在學校，把它改成了有介面的兩頁式程式</p>
<p>多虧自己在一開始的時候二二六六的研究了 python 的桌機介面寫法</p>
<p>雖然中間一度沒有使用</p>
<p>但最後和學弟討論後，馬上將之前的程式改一改</p>
<p>順便把我的程式優化整理一番</p>
<p>將本來需要三個 pyw 介面檔優化成只剩下一個</p>
<p>然後結合我自己中間用黑視窗寫的功能</p>
<p>一下就生出了現在的版本<img src="https://minayu0416.files.wordpress.com/2018/05/screen-shot-2018-05-24-at-2-12-25-pm.png" alt="Screen Shot 2018-05-24 at 2.12.25 PM"></p>
<p>沒錯，一個介面很陽春，但功能具備的小小桌機程式</p>
<hr>
<p>從原本完全矇懞懂懂，到跟學弟討論後</p>
<p>不只介面，功能生出來</p>
<p>而且還想出怎麼將我的程式優化</p>
<p>從原本需要透過初始頁面呼叫兩個介面及兩個外部副程式</p>
<p>到現在只需一個主程式外 call 一個副程式即可(當然還有伺服器端啦~)</p>
<p>而這週五，有個學弟想挑戰只需一個.pyw 檔就做到所有的事</p>
<p>在這記錄一下，pyw 檔怎麼打包才不會有黑視窗</p>
<p>請服用：</p>
<p>下 pyinstaller --nonconsole</p>
<p>指令就可以了!</p>
<hr>
<p>以下是整個開發過程的心得：</p>
<p>我的程式已經做好一版可以跑的了</p>
<p>除了這週五有學弟想挑戰再更優化我的程式外</p>
<p>就是一些我跑去人家電腦試，測出的一些例外 bugs，我需要改一下</p>
<p>然後就準備拿去給學生使用</p>
<p>然後我就要進入論文的下個階段了</p>
<p>也就是分析＋寫論文</p>
<hr>
<p>在一開始，其實我個人是很不想面對現實的，有逃避的狀況</p>
<p>然而當我們老師跟我說，我已經開天窗了</p>
<p>他又用了他那招，限你幾天內做出來的逼迫方式，來逼迫我趕出東西</p>
<p>那時候，我非常的緊張，就是被我們老師操，要在限時的時間內，操出東西</p>
<p>當然又開始了焦慮，尤其一緊張，手笨腦袋又打結</p>
<p>很慌很慌問了一堆人</p>
<p>後來去問老師</p>
<p>老師則是說，得嚇嚇你，不然你都不動！</p>
<hr>
<p>在這之後，我就放下心，穩穩慢慢的開發我的系統</p>
<p>因為我知道我自己什麼都不懂，要去重新研究勢必要花一點時間</p>
<p>也很感謝這陣子老師都沒催我</p>
<p>而是我自己因為看著日子一天天過，我自己很緊張</p>
<p>加上當初是閉門一個人寫，沒有人能夠討論，所以常常卡住，也常常很緊張，因為寫不出來</p>
<p>想到當初的恐懼，就覺得，慘了我會延畢</p>
<p>再加上當初家庭革命，我也就連續生病了有一個月吧，大大小小的疾病都出來。</p>
<p>後來，跟學弟們約，把握跟他們相處的時間</p>
<p>才短短一天，就把卡了好幾天的程式問題給解了</p>
<p>後來的幾週，我都會跟學弟們約</p>
<p>當然免不了請吃飯辣 XD</p>
<p>但除此之外，交換的條件還包括</p>
<p>他們幫我寫伺服器端，我幫他們寫他們被業主砲得很慘的 CSS</p>
<p>他們幫我寫副程式，我分析資料，幫他們 Cover 兩個月的薪水</p>
<p>當然除了這些交換條件外</p>
<p>交流了技術，例如怎麼做比較好</p>
<p>得到了很多建議以及如何解 Bugs</p>
<p>直到現在程式已經完成第一版</p>
<p>但尚在優化中</p>
<p>雖然跟當初的好幾個功能相比，只完成了兩大部分</p>
<p>臉部辨識跟滑鼠子系統是拿掉了</p>
<p>至於加分的部分，就是我進入下個階段要規劃的分析</p>
<p>在整個開發過程我學到的技術是</p>
<ul>
<li>Python Django Views 做為 HTTP 接收端，回傳資料回資料庫</li>
<li>Python Tkinter GUI 客戶端</li>
<li>Python 黑視窗 客戶端</li>
<li>不同的作業系統版本問題(win10 可以用，win7 不可用)</li>
<li>UX, UI</li>
<li>軟體測試</li>
</ul>
<p>除此之外，點出幾大項其他能力</p>
<ul>
<li>溝通能力</li>
</ul>
<p>我不知道外面的程式開發是單獨開發，還是團隊開發</p>
<p>因此，工程師本身也要具備優良的溝通能力</p>
<p>除了和其他工程師交流外，還要能跟老闆客戶交流</p>
<p>已暸解他們的需求</p>
<ul>
<li>解題(邏輯)能力</li>
</ul>
<p>會不會 google，要找什麼關鍵字，很重要</p>
<p>像我怎麼 google 都 google 不到</p>
<p>可是每次問學弟，拿學弟回的 google 都找得到 XD</p>
<p>還有就是資訊知識，通常技術都不會是問題。</p>
<p>問題最大的是，資料庫跟伺服器怎麼溝通，客戶端怎麼跟伺服器溝通</p>
<p>程式的話，只要會那幾個語法，其他上網找都找得到，但是技術知識不懂的話</p>
<p>就會像我這樣，很緊張一直到處亂問，桌機真的能丟資料到伺服器嗎？</p>
<p>這種蠢問題，我確實在剛開發得時候常常擔憂 XD</p>
<ul>
<li>程式能力</li>
</ul>
<p>程式能力還是重要啦～但只是基礎基礎肌基礎</p>
<p>雖然我現在 Python 也沒很強，但我桌機的部分，也是從開始開發才開始慢慢學 XD</p>
<p>也不是以前就會。</p>
<ul>
<li>穩穩地心情</li>
</ul>
<p>開發不能緊張不能慌，像我就很緊張很容易慌</p>
<p>但越慌越緊張腦袋越容易掛彩</p>
<p>我們動腦的工程師</p>
<p>常常花上一天卡同樣一個問題</p>
<p>也常常在除了寫程式之外的時間也還在思考剛剛那個問題我可以怎麼解</p>
<p>因此，絕對要保持自己腦袋營養，絕對不能掛彩 XD</p>
<p>最後，我就先記錄到這。</p>
<p>等待明天將程式完成後</p>
<p>下週應該就是，狂找電腦給人測試</p>
<p>之後就扔給學生蒐集資料了</p>
<p>然後開始進入下個階段了!</p>
