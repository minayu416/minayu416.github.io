<p>今天介紹 <code>__eq__, __hash__</code>在 Python 物件 Object 的做法</p>
<hr>
<p>起初是因為一連串重複的 object list 中取除獨特的物件 list</p>

<pre><code>>>> list = [2,3,4,5,5,5,5,2,2,3,4,5]
>>> list
[2, 3, 4, 5, 5, 5, 5, 2, 2, 3, 4, 5]
>>> set(list)
{2, 3, 4, 5}
</code></pre>
<p>所以代表 set 利用在數字上是可以去除重複值</p>
<p>假設我的物件 List 長這樣</p>
<pre><code>o_list = [data, data, data, data, data, data]
</code></pre>
<p>其中只會有 3 個物件是獨立的，但利用 set()去實作並沒有篩選出獨立的 3 個物件</p>
<p>我先後參考這些文件撰寫出程式碼</p>
<p><a href="https://stackoverflow.com/questions/3942303/how-does-a-python-set-check-if-two-objects-are-equal-what-methods-does-an-o">How does a Python set([]) check if two objects are equal? What methods does an object need to define to customise this?</a></p>

<p>然後造樣寫了一個這個</p>
<pre><code>class RemoveDuplicate(object):

    def __init__(self, data):

    def __eq__(self, other):

data_set = set()
for data, book in library:

<hr>
<p>來介紹一下此篇 Object, Equal, Hash</p>
<p>Equal 這個功能，前輩說在各大程式語言內都看得到，例如 JAVA  <a href="https://openhome.cc/Gossip/JavaEssence/ObjectEquality.html">openhome.cc</a> ，<a href="http://www.codedata.com.tw/java/object-equality-1/">CodeData</a></p>
<p>例如<a href="https://dotblogs.com.tw/onebin/2017/01/26/180419">C #</a></p>
<p>前輩丟了這篇給我讓我捉模一下  <a href="https://hynek.me/articles/hashes-and-equality/">Python Hashes and Equality</a></p>
<p>Equal 在這寫是檢查兩個物件有沒有相同</p>
<p>而相同的依據就是 <code>__eq__</code> 去定義</p>
<pre><code>def __eq__(self, other):
    return (self.id) == (other.id)
</code></pre>
<p>在我的程式碼內是使用 data.id 若相同就是同個物件</p>
<p>什麼 Hash? <a href="https://zh.wikipedia.org/wiki/%E6%95%A3%E5%88%97%E5%87%BD%E6%95%B8">雜湊函式</a></p>
<p>講一個例子就是有一副樸克牌，Hash 的元件為紅心，黑桃，方塊及梅花</p>
<p>接下來塞 52 張不同花色的牌卡下去後，經過 Hash，樸克牌會像牌七一樣，同樣花色的排一排，該是紅心的牌他就會排在紅心的 List 後</p>
<pre><code>紅心, [紅心Q, 紅心3, 紅心1, ...]

黑桃, [黑桃1, 黑桃K, 黑桃2,...]
</code></pre>
<p>Hash 算是將物件中的某幾個元件(自己定)建立為固定的指紋, 若資料符合一系列的指紋就會被識別出來</p>
<p>而且可被 Hash(Hashable)的元件要是不可變動(Immutable)，這樣才能拿來作 eq 比較</p>
<pre><code># 我用id 當hash
def __hash__(self):
    return hash(self.id)
</code></pre>
<p>所以<strong>eq</strong> + <strong>hash</strong> ，就是 hash 會篩選出不同的桶子，而 eq 則會去比對說不同桶子內若其中有什麼元件是相同的，就會判定兩個桶子(物件)是一樣的</p>
<p>根據 Python2.7<a href="https://docs.python.org/2/reference/datamodel.html#object.__hash__">文章</a>中，</p>
<p>要使用 Set 做 eq 比較，<strong>唯有被 hash 過的物件才會被放進 eq 比較</strong></p>
<p>若 Class Object 沒有<a href="https://docs.python.org/2/reference/datamodel.html#object.__eq__"><code>__eq__()</code></a>  的話，那就不用<a href="https://docs.python.org/2/reference/datamodel.html#object.__hash__"><code>__hash__()</code></a></p>
<p>可是如果有使用<a href="https://docs.python.org/2/reference/datamodel.html#object.__eq__"><code>__eq__()</code></a> ，就一定要用<a href="https://docs.python.org/2/reference/datamodel.html#object.__hash__"><code>__hash__()</code></a></p>
<p>意思是寫了 eq, 就要用 hash，而 set 會自動呼叫<a href="https://docs.python.org/2/reference/datamodel.html#object.__hash__"><code>__hash__()</code></a></p>
<p>來看一下 Python 3 與 2 的文件 <a href="https://docs.python.org/3.6/library/stdtypes.html#set">set 3.6</a>  及  <a href="https://docs.python.org/2/library/sets.html">set 2.7</a></p>
<pre><code>set 內的元件必須要可被Hash
The elements of a set must be &#x3C;span class="xref std std-term">hashable&#x3C;/span>.




set不能包含可變動的元件，像是list或字典。
但他可包含不可變動的群組，像是tuple
As a result, &#x3C;strong>sets cannot contain mutable elements&#x3C;/strong> such as lists or dictionaries.
However, they &#x3C;strong>can contain immutable collections&#x3C;/strong> such as tuples or instances of&#x3C;code class="xref py py-class docutils literal notranslate">&#x3C;span class="pre">ImmutableSet&#x3C;/span>&#x3C;/code>.
</code></pre>

<p>我找了一篇文章</p>

<h1 id="mutable-vs-immutable-objects-inpython"><a href="https://medium.com/@meghamohan/mutable-and-immutable-side-of-python-c2145cf72747">Mutable vs Immutable Objects in Python</a></h1>
<p>裡面提及</p>
<p>要使用 Set 及 <strong>eq</strong>和 <strong>hash</strong></p>
<p>首先確定 eq 的條件，接著</p>
<p>只有被 hash 的元件才會拿來做比較</p>
<p>set 會自動執行 hash, 若沒有 hash 函式就不會進到 eq</p>
<p>只有可被 hash(不可變動的元件 Immutable)，才可做為 eq 做比對</p>
<p>至於如果遇到字典, 串列，或者像我的例子中 OrderedDict'，真的要對這些變動元件做 hash 怎麼辦呢?</p>
<p>在我的 data 物件中，包含了<strong>OrderedDict</strong>, 而<strong>OrderedDict</strong>,屬於 Dictionary 的一種</p>
<p>Dictionary 是  <strong>Mutable objects</strong>, 不可被<strong>hash</strong></p>
<p><a href="https://stackoverflow.com/questions/15880765/python-unhashable-type-ordereddict">Python unhashable type: 'OrderedDict'</a></p>
<p>In your case, <code>var1</code> contains some object that is not hashable (it does not implement <code>hash()</code>).
This object is an <code>OrderedDict</code>, which is a <strong>mutable object</strong> and is <strong>not hashable</strong> by design.</p>
