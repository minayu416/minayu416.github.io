[後端小菜鳥] 軟體工程師的筆記本

December 3, 2018 minayu416

1 minute read

  • Twitter

  • Google+

  • Facebook

  • Reddit

  • LinkedIn

  • StumbleUpon

  • Pinterest

  • Email

生長在資訊科技迅速發展的年代，初始使用電腦，打開空白的記事本

陽春白色畫面，搭配簡簡單單的幾個小功能 －字型, 大小, 粗體, 斜體, 刪除線

進化到有各式各樣的文書軟體，使得純文字更多樣化

透過PPT, KeyNote… 等軟體編輯成一頁一頁活潑生動的故事

透過Word, Pages,.. 等寫作一篇又一篇的文章

對於時常在撰寫程式碼的軟體工程師們，也許已習慣面對一行一行有編碼的程式編譯器

鮮少打開文書軟體或筆記本去做紀錄, 像是程式的流程, 像是待辦事項… 等等，並不用撰寫於文書軟體上，且也無法達到該有的效用

常與程式編譯器一起工作的軟體工程師們，通常記事是於編譯器中打開一個新的檔案

並於一行一行整齊且有編號之中將事情記錄下來

MarkDown 在此時就出現了，我把他稱呼為工程師的筆記本

起始我是看著Senior 的前輩跟我講解Debug的流程

一樣打開一個New的程式編寫檔案，當紀錄事項時，卻運用了MarkDown(以下簡稱MD)語法

不同於我，還打開簡單的記事本寫下二二六六潦草不堪的段落文字，準備回座位後慢慢憑著記憶組織文章

MD 語法起初是我在GitHub上看到的.md檔使用

這裡有一些基本的用法

文書軟體其實是將電腦看得懂的程式語法包裝成人們看得懂的按鈕與動作，還多了很多花俏亮麗的功能

若需圖文並茂的做講解，我想利用現在的文書軟體都能達到很好的效果，但對於描述/記述程式或過程並不需要這麼花俏的表達

對於擅長寫程式的工程師來說，由於平時就在撰寫程式碼，所以僅需於搭配md 語法，並用於常常使用的編譯器中，也可達到美化的作用，而不光只是相同大小的文字

例如可使用

# H1
## H2
### H3

來代表文字標題大小

運用

- First item
- Second item
- Third item

可列出List

利用

- [ ] First item
- [ ] Second item
- [ ] Third item

也可當CheckBox

前輩將編寫好的.md檔 搭配 編譯器的 Preview 功能做使用

便能使原本死板板同樣大小的文字，瞬間變得有標題有描述/ 列表的整齊記事

除了.md檔可以當作程式操作手冊的撰寫外

於GitHub/ GitLab上也能使用md語法來開issue

特地發此文的是因為在最近接觸到的軟體 Slack 以及 Trello

也都支援MD語法，所以我認為這個記事類語法需要學起來

我想在未來, MD語法會越來越常被使用

在先前的文章中好像沒有提及，才特別寫一篇給MD
