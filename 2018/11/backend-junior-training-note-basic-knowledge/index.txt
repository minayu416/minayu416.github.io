9:T2063,<p>隨著資訊科技越來越發達</p>
<p>「程式語言」被媒體擴大宣傳，處處可見廣告及文宣，使得人人學 Code 時代來臨</p>
<p>為了追求「速成」，許多補習班/ 甚至有可能是學校教授，你的學長姐紛紛傳授了快速學成，追求效率的方法</p>
<p>為了履行研究所教授告訴我的「做中學，學中做」，我帶著僅僅做幾個小的網頁專案及什麼都會一點的經驗，投入了現在的工作</p>
<p>程式語言，甚至說是「資訊科技」，好入門卻難精，在這樣的過程中，「程式語言」往往只扮演著其中一個角色</p>
<p>在資訊系的學科中，「程式語言」只是其中必修(但擋修)的科目之一，資訊科技可說是包羅萬象，絕非僅有「程式語言」這麼簡單可概括</p>
<hr>
<p>還記得我兩年半前下定決心踏入資訊科技後，剛入學卻發現有著各式各樣的資訊領域</p>
<p>瞬間變得更茫然</p>
<p>每一個資訊領域又有著不同的知識與技能</p>
<p>第一步該做的就是選定想發展的領域</p>
<p>就我來說，我希望發展網頁設計</p>
<p>本身是學 Python 來寫網頁後端，在這條道路上目前正在經歷菜鳥新人時期</p>
<p>我本身的經驗如前面所說，雖然本科是資訊系但大學真的都在由我玩四年</p>
<p>研究所時期，利用了速成的方法，邊進行專案邊學習，還歷經了接不同專案，就學不同技能，造就了什麼都會一點但什麼都不精通</p>
<p>帶著淺淺的背景知識(還是從專案學的)，薄薄的 python 技能(可能只有 c 語言的 for 迴圈概念)，連 list, dict 是什麼都不知道</p>
<p>前端只會 css, html，不會用 git, docker 的情況下開始求職</p>
<p>看著目前現階段大部分公司會要求要有什麼樣的技能</p>
<p>稍微了解後，就直接上陣</p>
<p>以為到處會被打槍的情況下，發現其實還是有很多工作主動發出邀請面試</p>
<hr>
<p>新人的第一關就是俗稱找一家公司「練功」，有沒有前輩指導，我覺得還蠻重要的</p>
<p>多問，多學，不懂就問，被罵也問，不受批評不進步，不會導正之前的三觀</p>
<p>開始你會發現，自己速成出來離外面公司的要求有一個 GAP 橫溝，只好缺什麼補什麼</p>
<p>針對公司要求的專案，要有怎麼樣的知識/技能去完成任務</p>
<p>以及要有怎麼樣的基本功，能夠加快進步的腳步</p>
<p>接下來列出幾個基本功</p>
<ol>
<li>
<p>程式語言</p>
</li>
<li>
<p>背景知識</p>
</li>
<li>
<p>作業系統/環境架置</p>
</li>
<li>
<p>GitHub/ GitLab</p>
</li>
<li>
<p>google 能力 (哈哈 XD)</p>
</li>
</ol>
<p>在這之後的從新人突破慢慢進步到初階工程師</p>
<p>之前寫的<a href="/2018/11/%E5%BE%8C%E7%AB%AF%E5%B0%8F%E8%8F%9C%E9%B3%A5-%E5%A6%82%E4%BD%95%E7%AA%81%E7%A0%B4%E6%96%B0%E6%89%8B%E9%9A%8E%E6%AE%B5/">這篇</a>可以提供點方向</p>
<pre><code>程式語言
</code></pre>
<p>針對想前進的領域主要運用的程式語言學習</p>
<p>從開始求職到最近進公司兩三個月了，常常收到「你應該要買一本程式語言的書，重頭看到尾」</p>
<p>程式語言是工具，主要是解決問題，與電腦溝通的工具</p>
<p>既然得將它當作工具來解決每個問題，建構每個軟體，完成每個電腦的任務</p>
<p>運用自如，就得需要熟悉整個程式語言</p>
<p>當成為 Junior 工程師的瞬間，公司的前輩就會將你當作已經會使用工具了 XD，代表可以的話得在進公司前就先把程式語言練熟 XD</p>
<p><strong>刷 LeetCode</strong> 是一種方法(前輩提議)</p>
<p><strong>買本書從頭看到尾(實作到尾)</strong>，這已經有兩個以上的前輩跟我建議</p>
<p>除了自己本身學的程式語言外，還有牽扯到相關領域的背景知識也需要學習</p>
<p>打個比方，用 JAVA 學 JSP, JAVA 要熟以外，跟網頁有關的 SQL 語法也要一併學習</p>
<p>SQL , 還有分 Mysql, Sqlite , ... 各家有些微不同，還有版本上語法不同 XD</p>
<p>當邊做專案，邊出錯邊查邊學，整體效率就會變慢很許多 QQ</p>
<p>因此，當成為一個初階工程師前最好將程式語言運用自如</p>
<p>當然，如果不小心還沒打好基本功就進去公司的話,... 就期待後續發展了 xD</p>
<pre><code>背景知識
</code></pre>
<p>以做網頁來當 sample 的話</p>
<p>大架構的前後端，資料庫/後端及怎麼渲染前端中間的互動都要能熟悉</p>
<p>甚至更細節的專業背景，網路, Httpd ... 之類的知識也要知道</p>
<p>剛開始進公司的第一個任務是做效率的 API</p>
<p>我這個新手光是能將伺服器, 資料庫, 客戶端接起來就先惡補了好多知識</p>
<p>接下來居然要達成每秒輸出輸入達幾千次的效能問題</p>
<p>程式寫法, 連結的環節, 掉資料問題, debug 就讓我吃不消了 XD</p>
<p>前端網頁的框架, 怎麼與後端伺服器互動, ... 之類的，還有許多更多的背景知識，要多細節有多細節，雖然無法在工作前完全學習完畢，但基礎知識還是得先學著了解</p>
<pre><code>作業系統/環境架置
</code></pre>
<p>無論平時是使用 mac 還是 windows，建議<strong>Linux 系統</strong>一定要學</p>
<p>很多伺服器架設及伺服器端的運用都以 linux 系統為主</p>
<p>其中的<strong>shell</strong>指令，可以的話要先學起來</p>
<p>在此就奉上前輩推薦的<a href="http://linux.vbird.org/">鳥哥的 Linux 私房菜</a></p>
<p>學電腦資訊一定要會用 linux 呀!!</p>
<pre><code>GitHub/ GitLab
</code></pre>
<p>版本控制系統</p>
<p>如果不想如<a href="/2018/05/developpython-django-%E4%B9%8Brest-api%E9%96%8B%E7%99%BC5-server%E7%AB%AF-open-save-file-to-vm-cloud/">這篇</a>的我，因為一個失誤造成僅僅幾行卻花了幾天寫的程式碼缺失，一定要學會用版本控制 Git</p>
<p>且目前許多公司都已要求要會 git</p>
<p>我在研究所的時候都還不太會用，雖然現在進到公司裡，也還不太會使用就是 XD</p>
<p>但自己辛苦邊卡邊寫三週的程式碼，在一個失誤後全部都沒了</p>
<p>當下由於卡畢業進度，整個人近乎崩潰，嚇死</p>
<p>版本控制系統主要是 GitHub/GitLab 平台，除了其他強大的功能外</p>
<p>基本上就是將程式碼推上平台存起來(先簡單講)</p>
<p>每一次推上平台都需要將本次有新增/刪除/改寫的程式碼及檔案做一個 review，確定後才能上傳平台</p>
<p>且每次上傳都會有不同的版本號碼，代表每次的跟動</p>
<p>很有條理, 方便及效率的將開發程式的過程步驟分明</p>
<p>萬一遇到斷線，電腦被偷, 或者其他事情，程式都存在平台上</p>
<p>這邊記錄一些 git 指令</p>
<pre><code>git log -p
git push -f
git diff
git reset --hard


# 做 git squash，整併/更改所有已commit的動作
# 利用log 來找出要整併的分支的前一個動作版本碼
git log
# 下這個指令來進行squash 動作
git rebase -i [版本碼]

# ... 會有兩個步驟，都更改完儲存離開為執行





google能力 (哈哈XD)
</code></pre>
<p>報什麼 Error Log 就將關鍵字複製貼上 google 看能怎麼解決</p>
<p>還有遇到什麼問題，不會寫，或不確定也上 google 找到能初步執行成功的函式/小程式</p>
<p>然後再進行改寫，近一步的 Debug 學習可以參考<a href="/2018/11/%E5%BE%8C%E7%AB%AF%E5%B0%8F%E8%8F%9C%E9%B3%A5-%E5%A6%82%E4%BD%95%E7%AA%81%E7%A0%B4%E6%96%B0%E6%89%8B%E9%9A%8E%E6%AE%B5/">這篇</a>，應該有稍微解說學習 Debug 工具，還有<a href="/2018/11/%E5%BE%8C%E7%AB%AF%E5%B0%8F%E8%8F%9C%E9%B3%A5-debug-%E6%B5%81%E7%A8%8B/">這篇</a>是將前輩教導我的 Debug 遇到問題之流程解說記錄下來</p>
<p>前幾篇應該也都大部分寫 Debug 的重要與學習</p>
<p>遇到問題，或者不會寫時，先 google，真的查過沒找到，都不會了，在做詢問</p>
<p>也有幾個程式的交流網站可以分享／瀏覽 / 詢問</p>
<hr>
<p>那我先將基本功的資訊寫到這，後續有要補充或做更改再回來更改囉</p>
