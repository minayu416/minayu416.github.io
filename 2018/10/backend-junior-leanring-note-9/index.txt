<p>寫一個後台的函式有他的幾個步驟，要依照步驟去做處理</p>
</li>
</ol>
<p>通常一個後台的函式會有三個步驟</p>
<ol>
<li>
<p>從資料庫取得資料</p>
</li>
<li>
<p>對資料做處理</p>
</li>
<li>
<p>將資料 Return 回前端</p>
</li>
</ol>
<p>在第一個步驟可以先去判斷資料是有沒取到</p>
<ol>
<li>
<p>沒取到 >> 回傳空值</p>
</li>
<li>
<p>有取到 >> 回傳資料</p>
</li>
</ol>
<p>可以在這個步驟先判斷有沒有取到資料</p>
<p>再來第二個步驟：對資料做處理</p>
<p>在這個步驟會對資料做處理，所謂的例外案例函式</p>
<pre><code>Try:
Except:
</code></pre>
<p>絕大部分會用在這，這個部分也會出比較多錯也就是說</p>
<pre><code>要預期這個函式可能會出什麼錯，然後在函式裡先做處理
</code></pre>
<p>這個步驟要先把自己預期可能會發生的錯誤處理一次</p>
<p>真的不確定的才會包成 Try/Except</p>
<p>第三步驟：將資料 Return 回前端</p>
<p>這個步驟是導致我寫這些的原因哈哈 XD</p>
<p>因為我包成 Try/Except</p>
<p>可是取值是空值(系統卻還是把它當成值來計算)，導致我回傳給前端報錯</p>
<p>第三步驟重點是，回傳的型別要是正確的，不然傳給前端會噴錯誤</p>
<p>最後是第三大項</p>
