9:T16ae,<p>學了一陣子實作單元測試後, 先前是前輩會先把單元測試的架構寫好讓我去實作 code</p>
<p>但前輩走了之後，現在的單元測試已經無法符合新開發的功能，必須去研究新的方法來稍微改寫框架，這篇主要紀錄新學習的單元測試相關的功能 <code>動態生成單元測試</code> 及 <code>設置測試環境, 移除環境</code></p>
<p>我使用 IDE 為 <code>PyCharm</code>, 單元測試的套件為 <code>unittest</code>, 語言 <code>Python</code></p>
<h1>動態生成單元測試</h1>
<p>大部分撰寫單元測試的框架大致如下</p>
<pre><code class="language-py">from unittest import TestCase
class MyNewTest(TestCase):

    def test_function1(self):
        pass

    def test_function2(self):
        pass

    def test_function3(self):
        pass

</code></pre>
<p>做一個可被執行的測試函式，函式名稱開頭需為 <code>test_</code></p>
<p>通常這樣寫 IDE 很聰明的話會顯示 <code>綠色箭頭</code>，接著就可以點箭頭跑測試</p>
<p>不然就是在 terminal 下 command line 驅動單元測試</p>
<p>但是當我們有一些資料想要透過 for 迴圈 根據值來做測試該怎麼辦？</p>
<p>我起初的想法就是寫 for 迴圈測試</p>
<pre><code class="language-py">    def test_function(self):
        for a in data:
            result = get_result(a)
            assertEqual(a, b)

</code></pre>
<p>但這樣的方法就是萬一迴圈內的某筆 assert 測試錯誤，整個單元測試就停住，將不會再往下測！</p>
<p>可是，我想要知道我迴圈內丟進去每個資料測試的狀態，那可否動態生成單元測試？</p>
<p>可以，而且方法很簡單(但我用錯方法找錯方向卡了四天 qqq)</p>
<p>根據 Python <a href="https://docs.python.org/3/library/unittest.html">unittest 文件</a>，可以使用<code>subtest</code>來達成功能</p>
<p>您可以在網站內搜尋 <code>i=i</code> 來找到範例函式</p>
<pre><code class="language-py">class NumbersTest(unittest.TestCase):

    def test_even(self):
        """
        Test that numbers between 0 and 5 are all even.
        """
        for i in range(0, 6):
            with self.subTest(i=i):
                self.assertEqual(i % 2, 0)
</code></pre>
<p>這個範例是來驗證 <code>0 - 6</code> 每個數字是否可被<code>2</code>整除</p>
<p>那我們就來改寫一下</p>
<pre><code class="language-py">from unittest import TestCase
class MyNewTest(TestCase):

    def test_function(self):
        csv_path = "./test_data.csv"
        df = pd.read_csv(csv_path)
        for index, row in df.iterrows():
            # 可以根據值來命名單元測試的名稱唷！
            with self.subTest(row['title']):
                # 測試的code實作在這
                result = get_result(row['test_input'])
                assertEqual(expect, result)

</code></pre>
<p>如此一來 run 出的單元測試就可以根據想測的資料筆數來動態生成啦！</p>
<h1>設置測試環境, 移除環境</h1>
<p>另外一個是設置測試環境跟移除</p>
<p>因為測試要不留任何痕跡，而且執行單元測試時，需要先將測試功能的場景給佈置好</p>
<p>假設我有三個測試函式都需要設置環境跟移除環境</p>
<pre><code class="language-py">from unittest import TestCase
class MyNewTest(TestCase):
    # 只要繼承TestCase 就可以使用setUp, tearDown 他們是unittest內附有的功能
    def setUp(self):
        # 可以寫任何你想先設置環境的事情
        prepare(situation, value)

    def tearDown(self):
        # 結束後有沒有要刪除或者移除什麼
        os.remove("test.sqlite")

    def test_function1(self):
        pass

    def test_function2(self):
        pass

    def test_function3(self):
        pass

</code></pre>
<p>這樣的寫法每當我跑一個測試函式就會執行<code>setUp</code> 幫我佈置環境, 結束後會 <code>testDown</code> 移除環境</p>
<p>若整個 Class 一起執行會跑的順序是：</p>
<pre><code>setUp > test_function1 > tearDown >
setUp > test_function2 > tearDown >
setUp > test_function3 > tearDown
</code></pre>
<p>可是！！！ 就是有這個 <code>可是</code> !</p>
<p>若只想佈置一次測試環境就測全部的測試函式, 全部測試測完在移除環境呢？</p>
<p>我們可以使用 unittest 內附的 <code>setUpClass</code>, <code>tearDownClass</code> 來實現夢想 XD</p>
<p>這是在 unittest 套件內找到的方法</p>
<pre><code class="language-py">    @classmethod
    def setUpClass(cls) -> None: ...
    @classmethod
    def tearDownClass(cls) -> None: ...
</code></pre>
<p>所以剛剛的例子可以是:</p>
<pre><code class="language-py">from unittest import TestCase
class MyNewTest(TestCase):
    # 只要繼承TestCase 就可以使用setUpClass, tearDownClass
    @classmethod
    def setUpClass(cls):
        # 可以寫任何你想先設置環境的事情
        prepare(situation, value)

    @classmethod
    def tearDownClass(cls):
        # 結束後有沒有要刪除或者移除什麼
        os.remove("test.sqlite")

    def test_function1(self):
        pass

    def test_function2(self):
        pass

    def test_function3(self):
        pass

</code></pre>
<p>執行的順序就會是</p>
<pre><code>setUpClass > test_function1 > test_function2 >
test_function3 > tearDownClass
</code></pre>
<p>可以參考 <a href="https://stackoverflow.com/questions/23667610/what-is-the-difference-between-setup-and-setupclass-in-python-unittest/23670844">Stackoverflow</a> 的回答來得知 setup/teardown 的方法有什麼不同</p>
<p>你也可以進入 python unittest 的套件來了解哪寫函式功能可以使用</p>
