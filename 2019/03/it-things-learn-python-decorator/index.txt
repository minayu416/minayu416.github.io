9:T22e5,<p>在約莫一兩個月前，我被賦予一個任務是利用 Python Decorator 做重構</p>
<p>當時我還不知道使用 Decorator 來解決問題，可以砍掉專案一半的代碼</p>
<p>Decorator 是一個超棒的方法，但對於我來說它並不太好學習。</p>
<p>初次嘗試 Decorator 是在今年的 1 月，我用了剛跑完 Scurm 一個 Spring 結束的喘息時間，嘗試學習 Decorator，目的就是達成前輩開的最後一個 Issue: 用 Decorator 重構專案。</p>
<p>我有看過前輩使用 Decorator, 就像還在學日文 50 音的程度看著前輩們寫出一篇日文的文章，你看著前輩那些高超的程式碼，竟懷疑起自己到底能不能達到那個境界...</p>
<p>我東找找西找找就是沒辦法好好地寫出一個至少能動的 decorator</p>
<p>而這篇，就是寫紀錄，我成功學習了 Decorator 來重構專案程式碼，並刪除至少一半的 code</p>
<h3 id="什麼是-decorator">什麼是 Decorator</h3>
<p>Decorator 就是裝飾器, 如下列程式碼，很像 <code>裝飾</code> 在函式上的裝飾器</p>
<pre><code class="language-py">
@menu
def make_coffee():
    pass

@menu
def make_ice_tea():
    pass


</code></pre>
<p>在這個範例中 <code>@menu</code> 就是 Decorator，我簡單用做餐點的邏輯寫一個範例程式</p>
<p>我學會了 Decorator 並運用在專案，這讓我砍掉我先前寫的一半的程式碼，但在文章中我不會秀出我在專案寫的程式碼，會用很相同的邏輯改寫。</p>
<p>前提有提及在剛開始找遍各大網站就是看不懂 Decorator 怎麼使用 qq，但在功力提升的兩個月後，搭配找到的寶典，終於實作出我要的功能啦!!!</p>
<p><a href="http://ot-note.logdown.com/posts/67571/-decorator-with-without-arguments-in-function-class-form">寶典在此</a>，感謝大大的整理</p>
<h1 id="來實作-decorator-囉">來實作 Decorator 囉！</h1>
<p>這個實作參考 <code>有參數的 Decorator Function</code></p>
<p>再次強調這個參考實作來自<a href="http://ot-note.logdown.com/posts/67571/-decorator-with-without-arguments-in-function-class-form">OT Coding Note</a>，有興趣可以點進去看</p>
<pre><code class="language-py">Note: 需要多一層的 outter function 來傳入參數
 def decorateFruit(fruit, rotLevel):
    def outer_d_f(f):
        def d_f(*args, **kargs):
            print("%s %s before call" % (rotLevel, fruit))
            result = f(*args, **kargs)
            print("%s %s after call" % (rotLevel, fruit))
            return result
        return d_f
    return outer_d_f

@decorateFruit('banana', 'new')
def print_hello2():
    print("hello 2nd time.")

@decorateFruit('guava', '50% rot')
def print_hello3():
    print("hello 3th time.")
</code></pre>
<pre><code>> print_hello2()
> print('')
> print_hello3()
</code></pre>
<pre><code>output
new banana before call
hello 2nd time.
new banana after call

50% rot guava before call
hello 3th time.
50% rot guava after call
</code></pre>
<h3 id="改成自己想實現的功能">改成自己想實現的功能</h3>
<p>首先先介紹一下需求：</p>
<p>我在專案裡主要是運用來判斷眾資料庫欄位的 <code>資料型態</code>: object, float64 ..</p>
<p>所以如果有相關類似需求也可以拿來改寫！但由於是文章中就必須保密做改寫惹 qq</p>
<pre><code>需求描述：
假設我們開一個飲料店，做不同的飲料需要不同的材料(material)跟工具(tools)
每個飲料都會制定用哪些材料製作，製作的時候會檢查材料倉庫(storehouse)
若沒有足夠的材料可以製作飲料就不做，我要怎麼達成自動判斷呢？
</code></pre>
<p>我寫了一個範例如下，是改裝<a href="http://ot-note.logdown.com/posts/67571/-decorator-with-without-arguments-in-function-class-form">OT Coding Note</a> 的 <code>有參數的 Decorator Function</code></p>
<pre><code class="language-py"># 這是我的Decorator
def auto_check_material(material):
    def outer_func(make):
        def check_material(*args):
            # 假設 storehouse 是list, 內有倉庫全部有的做菜材料
            storehouse, tools = args
            # 檢查倉庫裡有沒有材料
            if material not in storehouse:
                # 倉庫裡沒材料所以 False 不做
                return False
            # 反之倉庫有材料, 我們就傳入製作飲料的function
            return make(*args)
        return check_material
    return outer_func

# 裝飾Decorator的函式們, 會輸入製作這杯飲料要具備什麼材料
@auto_check_material(['coffee beans', 'water'])
def make_coffee(food_storehouse, tools):
    # 倉庫有材料才會做
    pass

@auto_check_material(['black tea bag', 'water'])
def make_ice_tea(food_storehouse, tools):
    # 倉庫有材料才會做
    pass

@auto_check_material(['black tea bag', 'apple', 'water'])
def make_apple_tea(food_storehouse, tools):
    # 倉庫有材料才會做
    pass

</code></pre>
<p>有些註解寫好了</p>
<p>我的 Decorator 是 <code>auto_check_material</code>, 這個函式主要幫我自動確認倉庫內有無材料，傳入的變數為 <code>製作某個飲料需要的材料</code>。</p>
<p>會有三層 function (但我現在還沒深入研究 qq, 所以若講解怪怪的要自動訂正 XD)</p>
<p>合理假設 <code>outer_func(make)</code> 就是被裝飾的函式們, 也就是說我在 <code>check_material</code> 確認材料有在倉庫後就會執行 <code>return make(*args)</code> ，確認可以製作飲料，就進到製作的函式開始製作飲料。</p>
<h3 id="講解時間">＊＊＊講解時間＊＊＊</h3>
<p>我也是在打文章時才梳理清楚邏輯 XD, 也就是說這些函式的執行是</p>
<p>當程式一執行，最先會經過三個製作飲料的函式</p>
<p>假設一開始 run 遇到 make_coffee</p>
<pre><code class="language-py">
@auto_check_material(['coffee beans', 'water'])
def make_coffee(food_storehouse, tools):
    pass

</code></pre>
<p>接著發現製作咖啡 make_coffee 的函式上有個裝飾器 <code>auto_check_material</code>, 代表前提是要先通過<code>auto_check_material</code> 合格後才能執行 <code>make_coffee</code>。</p>
<p>裝飾器上帶有 <code>(['coffee beans', 'water'])</code>，這是執行 Decorator 需要用到的變數，在需求中代表著要有這兩個材料才能做咖啡。</p>
<p>進到 Decorator</p>
<pre><code class="language-py"># 這是我的Decorator
def auto_check_material(material):
    def outer_func(make):
        # 會先經過並執行check_material
        def check_material(*args):
            # 假設 storehouse 是list, 內有倉庫全部有的做菜材料
            storehouse, tools = args
            if material not in storehouse:
                return False
            # 若符合就進到outer_func也就是外部的 def make_coffee():
            return make(*args)
        return check_material
    return outer_func

</code></pre>
<p>因為進到裝飾器內，所以這邊的 <code>outer_func</code> 指的是在外層製作飲料的函式 <code>make_coffee</code></p>
<p>意思是說在這個裝飾器內，前提要先通過 <code>check_material</code> 確認材料都有在 <code>storehouse</code>內，才將變數 <code>*args</code> 塞往下一步 > <code>outer_func</code> 外部函式 (也就是咱製作飲料的函式)</p>
<pre><code class="language-py">
@auto_check_material(['coffee beans', 'water'])
def make_coffee(storehouse, tools):
    # 執行玩decorator後, 符合True 就做這個function
    # 變數 storehouse, tools 會一起送進Decorator 再回來這個函式中使用
    ...
    # 一連串製作後，生出咖啡，送出咖啡
    return coffee

</code></pre>
<p>執行完 Decorator >> outer_func >> 執行 make_coffee 製作咖啡</p>
<h3 id="總結順序">總結順序</h3>
<p>這邊釐清一下執行函式的順序</p>
<p>程式跑下去的順序依序是</p>
<pre><code>>> def make_coffee >> @auto_check_material >>
check_material(*args) >> outer_func(make) >> def make_coffee(製作)

>> def make_ice_tea >> @auto_check_material >>
check_material(*args) >> outer_func(make) >> def make_ice_tea(製作)

>> def make_apple_tea >> @auto_check_material >>
check_material(*args) >> outer_func(make) >> def make_apple_tea(製作)

</code></pre>
<p>嗯！大概是這樣子</p>
