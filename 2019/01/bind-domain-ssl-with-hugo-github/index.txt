9:T19ae,<p>約莫 3.4 天前，我穩定了我新架的 hugo 網站，把一些 bugs 修好後</p>
<p>終於可以慢慢邁向下一階段的目標</p>
<p>去年 12 月半，我決定動手架一個屬於自己的網站</p>
<p>以往都是想用 wordpress.org 架在主機上，但還是有些受限於 wordpress</p>
<p>於是我把想法移駕到了 hugo + github</p>
<p>某天，上網查查我新的部落格在 Google 的 SEO 是如何呈現</p>
<p>結果每篇文章的 Title 都掛上 "- Git 指令"</p>
<p>我來來回回煩惱後來問同事後，得到的解答是買一個自己的 domain</p>
<p>雖然知道遲早都會需要買，但沒想到現在就要買</p>
<p>目前我架站的方式是：</p>
<pre><code>Hugo 架設在Github 上，於Godaddy買一個Domain+SSL指向我的網站
</code></pre>
<p>接下來應該會邊陸續添增一些新的功能，寫文章, 慢慢提升 SEO</p>
<p>本文章教學：</p>
<h2>於 Godaddy 購買 domain</h2>
<p>Godaddy 網站  有許多 domain, 你可以直接去查看想要的 domain，順便看看這些 domain 的延伸含義</p>
<p>這個部分其實蠻簡單的</p>
<pre><code>將想要的domain查詢後 >> 加入購物車 >> 點取右上方購物車圖示 >>
填繳費表單 >> 勾選要不要額外的服務 >> 優惠碼 >> 結帳
</code></pre>
<p>特別提一下在購物車時，有個<code>網域隱私保護</code>, 其實只是讓你的 domain 能不能在 Whois365  中被查找，沒有網站說的什麼資訊看光光這麼可怕</p>
<p>但我還是被嚇唬到，然後購買了，好冤 XD</p>
<p>查看了一下 SSL 好像是額外買的，沒在這可以選</p>
<p>記得去領取優惠碼，據說可以打折(但我就算加了優惠碼也沒有打折氣噗噗)</p>
<h1>買 SSL Certification</h1>
<p>我也是在Godaddy SSL 購買 SSL</p>
<p>SSL 就是把你的網站從 Http 變為 Https</p>
<p>可惡的是購買 SSL 比購買網域貴好多!!</p>
<p>把工具買好惹，就來裝吧</p>
<h2>裝 SSL Certification 於 你的 domain</h2>
<p>Godaddy 的台灣版後台，我當初也是摸的翻白眼，不太好用，索性...... 用好了 qq</p>
<p>如果是單購買 Domain，然後將購買的 SSL 裝上去的話</p>
<p>在 Godaddy 登入你的帳號</p>
<p>前往後台左上方的 SSL 憑證</p>
<p>你會看到一條剛購買熱騰騰的 SSL 憑證，尚未設定</p>
<p>進入設定，會需要你輸入欲綁入的網址</p>
<p>(請原諒我沒有圖)</p>
<p>基本上會有左右兩塊，左邊是輸入你想認證 SSL 的 domain 網址，會產生密鑰，再將其中一個密鑰貼到右邊認證</p>
<p>輸入你想認證的 domian，會產生兩個檔案</p>
<p>假設我在 godaddy 買的 domain 是: mina.io</p>
<p>就直接輸入 mina.io, 不用加前面的 http/之類, 端看剛才於 Godaddy 買的 domain 是什麼就填什麼</p>
<p>會產生兩個檔案，分別是：</p>
<pre><code>generated-csr
generated-private-key
</code></pre>
<p>我有下載下來保存好</p>
<p>接下來，我記得是將產生的 generated-csr 貼到右方需要貼 csr 做認證的地方</p>
<p>在貼的部分下方，會再次詢問你是不是要認證該 domain?</p>
<p>確認後就送出吧</p>
<p>以下兩個比較複雜，促使我發出這篇文章 XD</p>
<p>把 SSL+ Domain 都搞定後，就來做連結了</p>
<h2>將 Domain 設定指向 Github</h2>
<p>打開 Goddady 的後台後，前往網域，打開三個點 ··· 進入 DNS 管理</p>
<p>接下來你會需要新增 在 <code>紀錄</code>裡</p>
<p>分別是 A 紀錄, CNAME</p>
<p>新增 A
首先你要找到你的 Github 的 A</p>
<p>參考Github 的網站 , 可以取得兩個指令來找到 Github 的 A</p>
<pre><code> $ dig [YOUR-USERNAME].github.io +noall +answer
</code></pre>
<p>將這段指令貼上你的 Terminal 來查找你 github 網站的 A</p>
<pre><code>[YOUR-USERNAME].github.io &#x3C; 改為[你的github帳號].github.io
</code></pre>
<p>＊注意：有幾個 A，在你的 Domain DNS 管理就要登幾個 A, 範例只有 1 個</p>
<pre><code>[YOUR-USERNAME].github.io     3600    IN A     199.27.XX.XXX
</code></pre>
<pre><code>[YOUR-USERNAME].github.io     3600    IN A     199.27.XX.XXX
</code></pre>
<pre><code>[YOUR-USERNAME].github.io     3600    IN A     199.27.XX.XXX
</code></pre>
<pre><code>[YOUR-USERNAME].github.io     3600    IN A     199.27.XX.XXX
</code></pre>
<p>有四筆，就要登入四筆</p>
<p>接著回到 Godaddy 的管裡 DNS 後台，「紀錄」右下方「加入」</p>
<p>新增這四筆</p>
<pre><code>類型 : A

主機: @

指向: port號

TTL 得符合上面的3600，也就是一小時 (看你Dig多少就填多少~)
</code></pre>
<h2>將 Github 設定指向 Domain</h2>
<p>在你的 Github 前往 你放網站的 repository</p>
<pre><code>[YOUR-USERNAME].github.io
</code></pre>
<p>進 settings 往下滑可以看到 custom domain</p>
<p>把你的 domain 貼到 custom domain 後按 save</p>
<p>這時候 Github 會在你的 repository 下新增一個你看不到的 CNAME 檔案</p>
<p>這邊我吃好多悶虧 qq</p>
<p>如果你回到 hugo, 利用 hugo 重新生 public 強制推上 Github</p>
<p>你要回到 settings 重新加 custom domain</p>
<p>如果刪掉 repository 重新創還是 404 或者沒畫面，無論做什麼事都 404/沒畫面，可能需要清除 Cookie (我是這樣才重新連回我的 Github.io)</p>
<p>若成功把 domain 與 github 連上，你會你的部落格內容，但 css/樣式／主題 會跑掉</p>
<p>這是因為連到新的 domain 了，沒有在 config.toml 更新</p>
<p>接著回到你的 Hugo, 把 Config.toml 檔</p>
<p>更改 baseURL</p>
<pre><code>baseURL = "https://your-domain"
</code></pre>
<p>hugo 一次 產生 public 推上 你的 repository</p>
<p>＊要注意一下如果強制推上 "git push -f" 要再回去 settings 新增 custom domain 才會產生 CNAME 檔案，才能接你的 domain</p>
<p>如果覺得每次 hugo 生成 public 強制推上，都會刪掉原先的 CNAME 檔案的話</p>
<p>就在你的 hugo web 資料夾的根部 (就是 config.toml 那層), 新增一個 CNAME 檔案(不用副檔名)</p>
<p>把你的 domain 直接寫在 CNAME 檔裡，儲存，之後每次下 hugo 生成 public 都會自動生成 CNAME 囉！</p>
<pre><code>CNAME
your-domain
</code></pre>
<p>＊只需要在第一行寫上你的 domain, 不用打 http/https 之類的</p>
<p>如果還是不知道要新增什麼，那就在 settings 新增 custom domain 後，把 repository git pull 下來，會看到新的一個 CNAME 檔案</p>
<p>那教學就到這囉，若有什麼問題可以來詢問我唷！</p>
