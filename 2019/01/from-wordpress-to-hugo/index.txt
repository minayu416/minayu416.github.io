<h2 id="找尋資料">找尋資料</h2>
<p>對於 Hugo 還一無所知，Hugo 與 Wordpress 這種 CMS 是不太一樣的系統

我前後參考了幾了網站來了解 Hugo 是一個 靜態網站生產器

一開始覺得很不靠鋪，我也是 wordpress 用戶，少了資料庫的靜態網頁生產器怎麼能媲美功能美滿的 CMS 呢?

後來我錯了, 架好 Hugo 後，我發現我可以離開 Wordpress 的限制，自由的改動網站的外觀，以及添加任何我想要的功能</p>
<p>非常的自由，但要有一個認知就是</p>
<p>所以我看了些比對的文章，像是選個最好的靜態網頁生產器, Jekyll vs Hugo

首先也要記得參觀Hugo 官網,選主題時你會常常在主題頁面中停留

用 Hugo 怎能不看 Hugo 官網呢?

再來我擔心沒有後台我就變成了不會寫文章的手殘

所以我找到了跟 Hugo 與 Github 相容的 CMS: Forestry

(可是我昨天實際用過 Forestry，不可否認很好用，但我現在卻正在調查她是不是造成我的網頁掛掉的兇手 ...，等我查明後才會推薦，這讓我的心情很鬱悶)

有機會分下個文章跟大家分享</p>
<h2 id="架設-hugo">架設 Hugo</h2>
<p>我找了一堆網站，來輔助我架設 Hugo</p>
<p>等到我真的開始實作後，我發現完全架不起來</p>
<p>這份簡報是我歷經千辛萬苦，架了幾十次，跟著架才成功的指南</p>
<p><code>讚</code></p>
<p><strong><a href="https://speakerdeck.com/fukuball/create-static-website-with-hugo-in-2-minutes">請服用</a></strong></p>
<p><code>我是使用Mac</code></p>
<h3 id="首先安裝-hugo">首先安裝 Hugo</h3>
<pre><code>$ brew install hugo
</code></pre>
<p>如果沒有 brew，可以試試看
<strong><em><a href="http://osxdaily.com/2018/03/07/how-install-homebrew-mac-os/">這篇文章</a></em></strong> 裝一下</p>
<p>＊註： <code>brew</code> 就是 mac 系統的套件管理系統 <code>Homebrew</code>, 若是沒有 homebrew，可以找尋安裝指南，幾行指令就搞定!</p>
<h3 id="新建一個網站">新建一個網站</h3>
<pre><code>hugo new site mysite
</code></pre>
<p>此時您的網站已經初始生成</p>
<h3 id="進入網站資料夾">進入網站資料夾</h3>
<pre><code>cd mysite
</code></pre>
<p>內部的資料夾包刮 content, theme, layout, ....</p>
<h2 id="安裝-theme">安裝 Theme</h2>
<p>接著就去 Hugo 官網挑挑你愛的主題吧!

我是挑 <strong><em><a href="https://themes.gohugo.io/hugo-tranquilpeak-theme/">Tranquilpeak</a></em></strong>

每個主題點進 Download 都會進到該主題的 Github

那我們就用 Git 指令把主題裝到我們的網站資料夾內</p>
<p>進到主題資料夾安裝</p>
<pre><code>$ cd themes
</code></pre>
<p>這邊以 Tranquilpeak 為例</p>
<pre><code>$ git clone https://github.com/kakawait/hugo-tranquilpeak-theme.git
</code></pre>
<p>裝好後利用 ls 指令會看到 hugo-Tranquilpeak-theme 資料夾在您眼前</p>
<pre><code>$ ls
hugo-Tranquilpeak-theme
</code></pre>
<p>接著看你要使用編譯器，還是手動拉</p>
<p>將一些相關文件從 theme 複製到你的根目錄</p>
<p><strong><em>將你的主題 theme 資料夾內 "archetyps" 內的檔案複製到你的"archetyps"</em></strong></p>
<p>這邊是用 hugo-Tranquilpeak-theme 主題</p>
<p>將<code>hugo-Tranquilpeak-theme/archetyps/</code>內容移動至 <code>mysite/archetyps/</code>內</p>
<p><strong><em>將你的主題 theme 資料夾內  exampleSite 內的檔案, 直接取代根目錄的檔案</em></strong></p>
<p>忘記在哪邊看到，在簡報中是沒有看到這個部分</p>
<p>要順利先將主題的頁面刻出來，就將 <code>exampleSite</code> 內像是 content, static 都直接複製取代掉你的<code>mysite/</code> 中的 content 及 static</p>
<p><strong><em>將你的主題 theme 資料夾內  exampleSite 內的 config.toml, 直接取代掉根目錄的 config.toml</em></strong></p>
<p>就是取代<code>mysite/config.toml</code> 中</p>
<p>接下來回到根目錄</p>
<p>將 hugo run 起來 就能看到完整的主題頁面</p>
<pre><code>$ hugo server
</code></pre>
<p>先求有再求好</p>
<p>接下來就是您的工作囉~</p>
<p>打開根目錄的 config.toml 檔案, 將參數改一改，改成你要的</p>
<h2 id="將-wordpress-文章輸出-轉為並放入-hugo">將 Wordpress 文章輸出, 轉為，並放入 Hugo</h2>
<p>當初要實作時，我參考了一個網站，發現完全不 work</p>
<p>我也實際參觀Hugo migrations的頁面，照著上面的一些功能，也是不 work xD</p>
<p>後來參照from wordpre to hugo 的文章後，使用了Exitwp ，才順利把文章從 xml 轉到 md 檔</p>
<h3 id="wordpress-後台-export-你的文章-xml">Wordpress 後台 Export 你的文章 .xml</h3>
<pre><code>[你的wordpress url]/wp-admin/
</code></pre>
<p>可以進到 wordpress 的管理員後台</p>
<pre><code class="language-">點選 Tools > Export
</code></pre>
<p>可以進去 Export 文章的網頁</p>
<p>＊若沒有 Export 要去 Plugin 安裝，wordpress.com 都會有，Wordpress.org 可能要安裝套件 Plugin</p>
<pre><code>選擇免費的 Start Export
</code></pre>
<p>開始把你的部落格點匯出，完成的檔案會寄信到您的信箱</p>
<h3 id="用外部套件將-wordpress-xml-轉為-hugo-使用-md">用外部套件將 Wordpress XML 轉為 Hugo 使用 md</h3>
<p>參考網站: from wordpre to hugo,  Exitwp</p>
<p>將 exitwp 下載下來</p>
<pre><code>git clone https://github.com/thomasf/exitwp.git
</code></pre>
<p>＊請注意這個套件的環境為 python 2.7</p>
<pre><code>pip install pip_requirements.txt
</code></pre>
<p>或者</p>
<pre><code>pip install
</code></pre>
<p>驅動 python 程式來轉檔</p>
<pre><code>python exitwp.py
</code></pre>
<p>幸運沒報錯的話，就可以進<code>build</code>資料夾領取 md 囉~</p>
<h3 id="將轉出的md-檔放進-hugo-內">將轉出的.md 檔放進 hugo 內</h3>
<p>這個部分老實說要看不同的主題會有不同的要求，主要看你的 網站資料夾 內的 "archetyps" / themes/你的主題/archetyps/內的.md 檔怎麼定義一篇部落格</p>
<p>將輸出的.md 檔放進 Hugo, 通常會是這幾個資料夾，取決於你的 theme/exampleSite/content 內容</p>
<pre><code>content/blog
content/post
content/posts
</code></pre>
<p>確認一下每篇 md 檔有沒有和 archetyps 內.md 規範符合</p>
<p>至少要有 archetyps 內.md 內規定的幾個元素</p>
<p>有可能是</p>
<pre><code>title
layout
</code></pre>
<p>或</p>
<p>最後，到根目錄跑一次建置，進入 1313 看一下網頁有沒有問題，就可以準備部署囉!</p>
<pre><code>hugo server
</code></pre>
<h2 id="部屬至-github">部屬至 Github</h2>
<p>將網頁建在哪都可以，AWS, GCP 或其他平台，而我看大部分的人都部屬在 github 上</p>
<p>這部分我自己做的也不是很好，沒辦法建議甚麼 qq</p>
<p>2019.05.19</p>
<p><strong><em>跟大家說個福音</em></strong>，那就是, 隨著我後來的功力大增，我終於知道怎麼好好的部署至 Github 啦！！！！</p>
<p>傳送門在此: <strong><em><a href="/2019/04/how-to-host-hugo-on-github/">[BLOG] 如何在 GITHUB 上架設 HUGO 網站</a></em></strong></p>
<p>當然還是可以繼續往下讀，只是當時不怎麼會弄寫得不怎麼好就是 XD</p>
<p>你可以參考hugo 本身的網站</p>
<p>但我用的方法:<a href="https://medium.com/@chs_wei/%E5%9C%A8-github-%E9%83%A8%E7%BD%B2-hugo-%E9%9D%9C%E6%85%8B%E7%B6%B2%E7%AB%99-9c40682dfe40">這位部落格主提供的方法</a></p>
<p>在 github 分成兩個 repo，一個是放你 build 網站的程式資料夾(當然你也可以選擇不存)</p>
<p>另外一個則是取名為</p>
<p>這個指令會生成 public 資料夾</p>
<p>接著再按照新開的 repo 上有指南，把 git init 後，往 github repo 推送</p>
<p>應該不用幾分後，進入你的網站</p>
<p>就可以看到你的網站囉!</p>
<p>除了注意一下環境要是 python2.7 外，若想使用 python3 執行</p>
<p>我想可以試試看在 exitwp.py 程式裡，把 print 換成 print() 語法</p>
<p>我有試過，但我忘了有沒有成功</p>
<p>＊exitwp 系統說缺少 yaml，可是我執行 pip install yaml 卻說沒有這個套件?</p>
<p>正確的套件是 PyYAML</p>
<p>＊theme 內只有 theme.toml 沒有 config.toml?</p>
<p>theme 內的 config.toml 在 範例網站的資料夾內</p>
<p>應該會有個 exampleSite 的 demo 網站\，裡面有 demo 用的 config.toml</p>
<p>＊為什麼我 hugo server 後畫面是空白的?</p>
<p>很多教學文章都是說將 theme clone 下來後，把 config.toml 檔改一改就好了</p>
<p>但實際上需要把 exampleSite 內的 archetyps, static 及 content 最好都取代根目錄的資料</p>
<p>(注意不要把 exampleSite 內的 content/post 內容取代掉你的文章)</p>
<p>確認主題可以顯示後，再把你的部落格文章.md 檔放進 content 內</p>
<p>＊為什麼我把我的文章放進 content 後跑不出來？</p>
<p>我想這要參考一下 theme 內的 archetyps 怎麼定義為一篇文章</p>
